# 슬라이드 프롬프트 생성기

연구 계획서 문서를 분석하여 핵심 개념을 추출하고, 각 개념에 적합한 시각화 레이아웃으로 이미지 생성 프롬프트를 작성합니다.

## 워크플로우

### Phase 1: 문서 분석

1. 입력 문서 읽기 (연구 계획서, 사업 계획서 등)
2. 핵심 개념 추출:
   - 비전/목표
   - 세부 기술/기능
   - 핵심 메커니즘
   - 단계별 계획
   - 추진체계

3. 슬라이드 구성 계획:
   - 추출된 개념별 슬라이드 제목 결정
   - 각 슬라이드에 적합한 레이아웃 유형 선택

### Phase 2: 레이아웃 선택

[references/layout_types.md](references/layout_types.md) 참조하여 개념에 맞는 레이아웃 선택:

| 개념 유형 | 레이아웃 |
|:---|:---|
| 비전, 목표 계층 | 계층형 피라미드 |
| 통합 플랫폼, 범용 시스템 | 중앙 방사형 |
| 데이터 파이프라인, 순환 프로세스 | 원형 순환 |
| 하드웨어 모듈, 구성품 | 3D 분해도 |
| 실제 vs 가상, 전후 비교 | 좌우 병렬 비교 |
| 연도별 계획, 로드맵 | 수평 타임라인 |
| 조직 구조, 협력 관계 | 조직도 + 네트워크 |
| 분류 체계, 매트릭스 | 표 구조 |

### Phase 3: 프롬프트 작성

[references/prompt_style_guide.md](references/prompt_style_guide.md)의 스타일 가이드 적용.
[assets/output_template/prompt_template.md](assets/output_template/prompt_template.md) 템플릿 기반으로 작성.

프롬프트 구조:
```
1. 목적 (1-2문장)
2. 전체 톤앤매너
3. 스타일
4. 색상 (HEX 코드 포함)
5. 조명/해상도
6. 슬라이드 레이아웃
7. 상단 타이틀
8. [메인 콘텐츠 섹션들]
9. 인포그래픽 디테일
10. 금지 요소
11. 최종 결과물 목표
```

### Phase 4: 출력

생성된 프롬프트를 `prompt_image_gen/` 내 프롬프트 폴더 생성하여 저장 또는 사용자 지정 폴더에 저장.
파일명 형식: `[번호]_[슬라이드제목].md`

## 공통 규칙

- 텍스트: 한글 기본, 기술 용어는 영어 병기
- 스타일: 정돈된 3D 렌더링 + 미니멀 인포그래픽
- 시점: 약한 Isometric
- 금지: 만화풍, SF/사이버펑크, 어두운 배경
- 해상도: 8K Quality
