# 슬라이드 레이아웃 유형

개념 유형에 따라 적절한 시각화 레이아웃 선택.

---

## 1. 계층형 피라미드

**적합한 개념**: 비전/목표, 단계별 발전, 우선순위 구조

```
구조:
- 최상단: 최종 비전 (가장 좁음, 딥 블루)
- 중단: 상위 목표 (중간 너비, 미디엄 블루)
- 하단: 기반 기술/단계 (가장 넓음, 라이트 블루)

시각화:
- 색상 그라데이션 (하단→상단: 연한→진한)
- 층 간 구분선
- 위쪽 방향 화살표 (미묘하게)
```

---

## 2. 중앙 방사형 (Hub-Spoke)

**적합한 개념**: 통합 플랫폼, 1:N 관계, 중앙 집중 시스템

```
구조:
- 중앙: 핵심 노드 (가장 크고 진한 색상)
- 외곽: 연결된 요소들 (균등 배치, 72도 간격 등)
- 연결선: 중앙에서 외곽으로 방사

시각화:
- 중앙 노드 강조 (크기, 글로우)
- 데이터 흐름 표현 (점선 이동)
- 양방향 또는 단방향 화살표
```

---

## 3. 원형 순환 (Flywheel)

**적합한 개념**: 파이프라인, 반복 프로세스, 피드백 루프

```
구조:
- 중앙: 허브 (플랫폼/코어)
- 외곽: 순환 단계 노드 (72도 간격)
- 화살표: 시계 방향 순환

시각화:
- 각 단계별 고유 색상
- 회전 효과 (가속 느낌)
- 단계 간 곡선 화살표
- 외부 입력/출력 표시
```

---

## 4. 3D 분해도 (Exploded View)

**적합한 개념**: 모듈 구조, 하드웨어 구성, 조립 과정

```
구조:
- 중앙: 본체 (완성품)
- 분리: 구성 요소들 (본체에서 떨어진 위치)
- 점선: 장착 위치 연결

시각화:
- 약한 Isometric 시점
- 모듈별 고유 색상
- 깊이감 유지하며 분리
- 장착 전/후 비교 가능
```

---

## 5. 좌우 병렬 비교

**적합한 개념**: 전후 비교, 실제 vs 시뮬레이션, 대비 표현

```
구조:
- 좌측: A 상태/환경
- 우측: B 상태/환경
- 중앙: 변환/연결 요소

시각화:
- 동일한 구도/앵글
- 색상으로 구분 (예: 자연색 vs 블루 톤)
- 양방향 화살표 (변환 관계)
- 대응 요소 시각적 연결
```

---

## 6. 수평 타임라인

**적합한 개념**: 로드맵, 연도별 계획, 단계별 진행

```
구조:
- 상단: 연도 축 (좌→우: 시간 흐름)
- 하단: 트랙별 마일스톤 노드
- 구분: 단계별 색상 배경

시각화:
- 연도 마커 (원형 노드)
- 트랙별 고유 색상
- 핵심 마일스톤 ★ 표시
- 단계 경계 명확화
```

---

## 7. 조직도 + 네트워크 혼합

**적합한 개념**: 추진체계, 협력 관계, 역할 분담

```
구조:
- 상단: 총괄 기관 (가장 큰 노드)
- 중단: 세부 과제/부서 (분기 구조)
- 하단: 참여 기관/역할 (네트워크 연결)

시각화:
- 계층 관계: 굵은 실선
- 협력 관계: 얇은 점선
- 기관 유형별 색상
- 예산/역할 라벨
```

---

## 8. 표(Matrix) 구조

**적합한 개념**: 분류 체계, 비교 분석, 다차원 정보

```
구조:
- 열(Column): 분야/카테고리
- 행(Row): 기능/레벨
- 셀: 각 조합의 내용

시각화:
- 행 간 대비 (통일 vs 차별화)
- 열 간 구분 (색상 밴드)
- 헤더 아이콘
- 셀 내 간단한 시각 요소
```

---

## 레이아웃 선택 가이드

| 개념 유형 | 권장 레이아웃 |
|:---|:---|
| 비전, 목표 계층 | 계층형 피라미드 |
| 통합 플랫폼, 범용 AI | 중앙 방사형 |
| 데이터 파이프라인 | 원형 순환 |
| 하드웨어 모듈 | 3D 분해도 |
| 실제 vs 시뮬레이션 | 좌우 병렬 비교 |
| 연도별 계획 | 수평 타임라인 |
| 조직 구조 | 조직도 + 네트워크 |
| 분류 체계 | 표 구조 |
