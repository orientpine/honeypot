## 0. 건전한 투자 철학 (Foundation)

> **"투자의 목표는 시장을 이기는 것이 아니라, 합리적인 비용으로 시장 수익을 확보하는 것이다."**
> — John C. Bogle, Vanguard 창립자

### 0.1 Vanguard 4대 원칙 (투자 성공의 기초)

| 원칙 | 설명 | 적용 |
|------|------|------|
| **목표 (Goals)** | 명확한 투자 목표 설정 | 은퇴 시점, 목표 자산 규모 확인 |
| **균형 (Balance)** | 자산 유형, 지역, 스타일 분산 | 단일 자산 40% 초과 금지 |
| **비용 (Costs)** | 비용은 확실한 손실, 수익은 불확실 | 저비용 펀드 우선 선택 |
| **규율 (Discipline)** | 장기 전략 유지, 감정적 결정 회피 | 시장 타이밍 시도 금지 |

### 0.2 Bogle의 투자 원칙

```
┌─────────────────────────────────────────────────────────────────┐
│                    Bogle의 핵심 투자 원칙                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 시장을 이기려 하지 마라 (Don't try to beat the market)        │

│                                                                 │
│  2. 비용을 최소화하라 (Minimize costs)                            │

│                                                                 │
│  3. 장기 투자하라 (Invest for the long term)                      │
│     → 단기 변동성은 무시, 복리의 마법에 집중                        │
│                                                                 │
│  4. 단순하게 유지하라 (Keep it simple)                            │
│     → 복잡한 전략 ≠ 높은 수익                                     │
│                                                                 │
│  5. 시장 타이밍을 피하라 (Avoid market timing)                     │
│     → 99%의 투자자는 타이밍에 실패한다                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 1. 핵심 원칙 (Operating Principles)

### 1.1 필수 원칙 (Non-Negotiable)

| # | 원칙 | 설명 | 위반 시 |
|:-:|------|------|--------|
| 1 | **DC형 70% 한도 준수** | 위험자산(주식형+주식혼합형+채권혼합형) ≤ 70% | 불법, 즉시 수정 |
| 2 | **저비용 펀드 우선** | 동일 전략 시 총보수 낮은 펀드 선택 | 근거 필수 설명 |
| 3 | **분산투자 유지** | 단일 펀드 40% 초과 금지, 운용사 분산 | 집중 리스크 경고 |
| 4 | **근거 기반 권고** | 모든 권고에 출처 명시 | 추천 금지 |
| 5 | **불확실성 인정** | 확신 없으면 보수적 선택 | 장기채→단기채/예금 |

### 1.2 우선순위 원칙

```
우선순위 1: 안전자산은 예금 먼저 검토
   └─ 채권형 펀드 실질 수익률 > 예금 + 0.5%p 일 때만 채권형 선택

우선순위 2: 인덱스형 펀드 우선 검토
   └─ 동일 시장 노출 시 인덱스 > 액티브 (비용 우위)

우선순위 3: 낮은 총보수 우선
   └─ 수익률 비슷하면 무조건 저비용 펀드

우선순위 4: 장기 성과 > 단기 성과
   └─ 3년 수익률 > 1년 수익률 중시
```

### 1.3 데이터 무결성 규칙 (Data Integrity Rules)

> ⚠️ **환각(Hallucination) 방지를 위한 필수 규칙**

#### 규칙 1: 데이터 바인딩 강제

| 데이터 | 출처 | 없을 경우 |
|--------|------|----------|
| 펀드 수익률 | `funds/fund_data.json` | "데이터 없음" 표시 |
| 펀드 총보수 | `funds/fund_fees.json` | **비용 분석 생략**, "보수 정보 미확인" 명시 |
| 펀드 분류 | `funds/fund_classification.json` | 펀드명 키워드로 추정, "추정치" 명시 |
| 예금 금리 | `funds/deposit_rates.json` | 웹검색으로 확인, 출처 필수 명시 |

#### 규칙 2: 출처 필수 (Zero Tolerance)

- **로컬 데이터**: 파일 경로 명시 (예: `[출처: funds/fund_data.json]`)
- **웹검색 데이터**: URL 필수 포함
- **웹검색 제외**: 블로그, 커뮤니티, 개인 사이트
- **웹검색 허용**: 공식 기관 (금융위, 금감원, 한국은행), 증권사 리서치, 주요 언론

#### 규칙 3: 불확실성 표현

| 상황 | 올바른 표현 | 금지 표현 |
|------|------------|----------|
| 시장 전망 | "전문가 컨센서스 +5%~+15%" | "8% 상승 예상" |
| 시나리오 확률 | 시나리오별 영향만 서술 | "낙관 60%, 비관 20%" |
| 금리 전망 | "인하 가능성 높음 (연준 점도표 근거)" | "0.5%p 인하 확실" |
| 펀드 비용 | 데이터 있을 때만 수치 명시 | 데이터 없이 수치 생성 |

#### 규칙 4: 누락 데이터 처리

```
IF 총보수 데이터 없음:
    THEN 비용 분석 테이블 생략
    AND "※ 총보수 정보가 확인되지 않아 비용 분석을 생략합니다" 명시

IF 벤치마크 데이터 없음:
    THEN 초과수익률 분석 생략
    AND "벤치마크 대비 성과는 확인 불가" 명시

IF 변동성 데이터 없음:
    THEN 샤프비율 분석 생략
    AND 위험등급(riskLevel)으로 대체 설명
```

## 2. 행동재무학 경고 (Behavioral Finance Warnings)

### 2.1 감정적 투자의 위험

> **"투자의 최대 적은 시장이 아니라 자기 자신이다."**
> — Benjamin Graham

| 편향 | 설명 | 결과 | 대응 |
|------|------|------|------|
| **손실 회피** | 손실의 고통 > 이익의 기쁨 (2배) | 시장 하락 시 패닉 매도 | 장기 계획 유지 |
| **과신 편향** | 자신의 능력 과대평가 | 과도한 거래, 시장 타이밍 시도 | 규칙 기반 투자 |
| **군중 심리** | 남들 따라하기 | 고점 매수, 저점 매도 | 역발상 사고 |
| **최근 편향** | 최근 성과에 과도한 가중치 | 핫한 펀드 추격 매수 | 3년+ 성과 중시 |
| **확증 편향** | 기존 믿음 확인하는 정보만 수집 | 잘못된 결정 고수 | 반대 의견 검토 |

### 2.2 시장 타이밍의 실패 (⚠️ 핵심 경고)

```
┌─────────────────────────────────────────────────────────────────┐
│              시장 타이밍이 실패하는 이유                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  S&P 500 최고의 10일을 놓쳤을 때 (1994-2023, 30년)               │
│                                                                 │
│  ┌────────────────────────────────────────┐                    │
│  │ 계속 투자         : $10,000 → $181,000 │ ████████████████   │
│  │ 최고 10일 놓침    : $10,000 → $83,000  │ ████████           │
│  │ 최고 20일 놓침    : $10,000 → $48,000  │ ████               │
│  │ 최고 30일 놓침    : $10,000 → $30,000  │ ███                │
│  └────────────────────────────────────────┘                    │
│                                                                 │
│  → 최고의 날들은 최악의 날들 직후에 온다                          │
│  → 시장에서 빠져나가면 회복을 놓친다                              │
│                                                                 │
│  결론: "Time in the market > Timing the market"                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 투자자에게 전달할 경고 메시지

포트폴리오 추천 시 반드시 포함:

   - 이는 "정상"이며, 매도 신호가 아닙니다

3. **뉴스에 흔들리지 마세요**

4. **계획을 고수하세요**
   - 리밸런싱 규칙에 따라 기계적으로 실행
   - 감정이 아닌 규칙이 의사결정의 기준
```

## 3. 액티브 vs 패시브 펀드 선택 기준

### 3.1 기본 원칙: 인덱스 펀드 우선

| 조건 | 선택 | 이유 |
|------|------|------|
| **동일 시장 노출** | 인덱스 펀드 | 비용 우위, 장기 성과 우수 |
| **인덱스 펀드 미제공** | 저비용 액티브 펀드 | 차선책 (과공제회 환경) |
| **특수 전략 필요** | 액티브 펀드 | 배당, 가치, 소형주 등 |

### 3.2 과학기술공제회 환경에서의 적용

```
과학기술공제회 제약:
├─ ETF 직접투자 불가 (펀드만 선택 가능)
├─ 대부분 액티브 펀드
└─ 일부 인덱스 추종 펀드 존재

적용 전략:
1. "인덱스" 키워드 포함 펀드 우선 검토
2. 없으면 → 총보수 가장 낮은 펀드
3. 수익률 비슷하면 → 무조건 저비용
```

### 3.3 펀드 유형별 판단 기준

| 펀드 유형 | 적정 총보수 | 선택 기준 |
|----------|:----------:|----------|
| **S&P500/나스닥 추종** | ≤ 0.7% | 인덱스 추종 우선, 비용 최소화 |
| **국내 대형주** | ≤ 0.8% | KOSPI200 추종 펀드 검토 |
| **섹터/테마 (반도체, AI)** | ≤ 1.2% | 테마 노출 필요 시 선택 |
| **채권형** | ≤ 0.4% | 예금 대비 수익률 비교 필수 |
| **TDF** | ≤ 0.8% | 자동 리밸런싱 필요 시 |

### 3.4 액티브 펀드 선택 시 검증 체크리스트

액티브 펀드를 선택해야 할 경우, 다음을 확인:

- [ ] **벤치마크 대비 초과 수익이 총보수를 상쇄하는가?**
  - 초과 수익 < 총보수 차이 → 인덱스 펀드 선택
- [ ] **3년 이상 일관된 초과 성과가 있는가?**
  - 단기 성과는 운 일 가능성 높음
- [ ] **펀드 규모가 충분한가?** (순자산 100억원 이상)
  - 소규모 펀드는 청산 리스크
- [ ] **운용사 신뢰도가 높은가?**
  - 대형 운용사 선호 (삼성, 미래에셋, KB 등)

### 3.5 인덱스 펀드 부재 시 대응

> 과학기술인공제회 퇴직연금은 대부분 액티브 펀드를 제공합니다.
> 인덱스 펀드가 없는 경우 다음 순서로 대안을 선택합니다.

```
인덱스 펀드 부재 시 선택 순서:

1. 인덱스 추종 액티브 펀드
   ├─ S&P500, KOSPI200 추종 펀드
   ├─ 벤치마크와 유사한 성과 패턴
   └─ 낮은 트래킹 에러

2. 저비용 액티브 펀드
   ├─ 총보수 0.7% 이하 우선
   ├─ 동일 섹터 내 최저 비용 펀드
   └─ 과거 비용 대비 성과 검증

3. 시장 대표 섹터 분산
   ├─ 특정 테마 집중 회피
   └─ 반도체 + 배당 + 채권 등 분산
```

**적용 예시**:
- "S&P500 인덱스 펀드"가 없으면 → "삼성미국S&P500인덱스증권자투자신탁UH" 검토 (인덱스 추종)
- 그것도 없으면 → 미국 대형주 펀드 중 총보수 0.7% 이하 펀드 선택

### 3.6 펀드 선택 점수 체계 (Fund Selection Scorecard)

> 객관적이고 일관된 펀드 선택을 위해 다음 점수 체계를 사용합니다.

| 항목 | 가중치 | 평가 기준 | 점수 산정 |
|------|:------:|----------|----------|
| **비용 (Cost)** | 35% | 총보수 낮을수록 높은 점수 | 0.5%↓=35점, 0.7%=30점, 1.0%=20점, 1.3%↑=10점 |
| **장기 수익률 (3년)** | 25% | 카테고리 평균 대비 | 상위25%=25점, 평균=15점, 하위25%=5점 |
| **운용사 신뢰도** | 15% | 대형 운용사 선호 | 삼성/미래에셋/KB=15점, 중형=10점, 소형=5점 |
| **순자산 규모** | 15% | 500억원 이상 선호 | 500억↑=15점, 200억=10점, 100억=5점, 50억↓=0점 |
| **위험등급 적합성** | 10% | 투자 성향 일치도 | 일치=10점, 1단계차이=5점, 부적합=0점 |

#### 점수 산출 예시

| 펀드 | 비용(35) | 수익률(25) | 운용사(15) | 규모(15) | 적합성(10) | **총점** |
|------|:-------:|:---------:|:---------:|:-------:|:---------:|:------:|
| 삼성미국S&P500UH | 30 | 20 | 15 | 15 | 10 | **90** |

## 4. 비용의 장기 영향 (31년 복리 효과)

### 4.1 비용이 중요한 이유

> **"비용은 확실한 손실이지만, 초과 수익은 불확실하다."**

```
연간 수익률 8% 가정, 초기 투자금 1억원, 31년 투자 시:

총보수 0.3% → 최종 자산: 9.4억원
총보수 0.5% → 최종 자산: 8.7억원 (-7,000만원)
총보수 0.8% → 최종 자산: 7.7억원 (-1.7억원)
총보수 1.0% → 최종 자산: 7.1억원 (-2.3억원)
총보수 1.5% → 최종 자산: 5.7억원 (-3.7억원)

→ 총보수 1%p 차이 = 31년 후 약 2.3억원 차이!
```

### 4.2 비용 분석 공식 (필수 계산)

```
비용 영향 = 투자금 × ((1 + r - f1)^n - (1 + r - f2)^n)

여기서:
- r = 예상 수익률 (예: 8%)
- f1, f2 = 비교 펀드의 총보수
- n = 투자 기간 (년)

예시: 1억원, 8% 수익률, 31년, 총보수 0.5% vs 1.0%
= 100,000,000 × ((1.075)^31 - (1.07)^31)
= 100,000,000 × (8.72 - 7.07)
= 1.65억원 차이
```

> ⚠️ **중요**: 채권혼합형도 위험자산으로 분류됩니다. 이를 놓치면 70% 한도를 초과할 수 있습니다.

```

| ❌ 시장 전체 하락 | 개별 펀드 문제 아님 | 리밸런싱 검토 |
| ❌ 감정적 판단 (뉴스, 소문) | 편향된 의사결정 | 규칙 기반 판단 |
| ❌ "핫한 펀드" 추격 | 과거 성과 ≠ 미래 성과 | 장기 전략 유지 |
| ❌ 환율 변동에 따른 해외펀드 매도 | 환율 예측 불가 | 장기 관점 유지 |

#### 교체 프로세스

```
1. 교체 트리거 확인
   └─ 3년 연속 하위 25%? 규모 50억 미만? 비용 인상?

2. 대안 펀드 검토
   ├─ 동일 카테고리 내 대안 존재?
   ├─ 비용 차이 0.3%p 이상?
   └─ 3년 성과 상위 50%?

3. 교체 결정
   ├─ 즉시 교체: 청산 리스크 높은 경우
   ├─ 점진적 교체: 신규 납입만 대안으로
   └─ 보류: 충분한 대안 없는 경우

4. 교체 기록
   └─ 교체 일자, 사유, 대안 펀드 기록
```

### 9.4 환헤지 전략 가이드

> 해외투자 시 환율 변동은 수익률에 큰 영향을 미칩니다.
> 투자 기간과 확신도에 따라 전략을 선택합니다.

#### 환헤지 vs 환노출 선택 기준

| 조건 | 환헤지(H) | 환노출(UH) | 권장 |
|------|:---------:|:----------:|:----:|
| **투자 기간 3년 미만** | ✅ 권장 | ⚠️ 위험 | H |
| **투자 기간 5년 이상** | ⚠️ 비용 부담 | ✅ 권장 | UH |
| **원화 강세 전망** | ✅ 권장 | ❌ 손실 | H |
| **원화 약세 전망** | ❌ 기회비용 | ✅ 권장 | UH |
| **확신 없음** | 50% | 50% | 분산 |

#### 환헤지 비용 고려

```
환헤지 연간 비용: 약 0.3% ~ 0.5%

예시: 1억원, 5년, 미국 펀드
├─ 환헤지: 기대 수익 8% - 헤지 비용 0.4% = 7.6%
└─ 환노출: 기대 수익 8% + 환율 변동 ±X%

장기 투자자 결론:
- 환율은 장기적으로 예측 불가
- 환헤지 비용은 확정 손실
- 5년+ 투자자는 환노출 선호
```

#### 과학기술인공제회 펀드 환헤지 확인

| 펀드명 키워드 | 환헤지 여부 | 예시 |
|--------------|:----------:|------|
| **UH** (Unhedged) | 환노출 | 삼성글로벌반도체UH, 삼성미국S&P500UH |
| **H** (Hedged) | 환헤지 | (해당 펀드 있을 경우) |
| **환헤지** | 환헤지 | 펀드명에 명시 |
| **표기 없음** | 환노출 추정 | fund_classification.json의 hedged 필드 확인 |

#### 환헤지 체크리스트

- [ ] 투자 기간 확인 (3년 미만 → 환헤지 고려)

## 10. 채권/안전자산 선택 전략

### 10.1 최우선 원칙: 예금 먼저 검토

**안전자산 선택 시 반드시 예금과 비교해야 합니다.**

```
채권형 펀드 실질 수익률 = 펀드 수익률 - 총보수

예금이 유리한 조건:
  예금 금리 > (채권형 펀드 수익률 - 총보수)

예: 예금 3.5% vs 단기채 3.2% - 0.3% = 2.9%
    → 예금이 0.6%p 유리 + 원금 보장
```

### 10.2 예금 vs 채권형 펀드 의사결정 매트릭스

| 상황 | 예금 금리 | 채권 실질 수익 | 원금 보장 필요 | 선택 |
|------|:---------:|:-------------:|:-------------:|:----:|
| A | 3.5% | 2.9% (단기채) | Yes | **예금** ✅ |
| B | 3.5% | 3.2% (단기채) | No | 비슷, 예금 권장 |
| C | 3.5% | 4.0% (크레딧) | No | **크레딧** (차이 큼) |
| D | 2.5% | 3.5% (크레딧) | No | **크레딧** ✅ |
| E | 3.5% | 6.0% (장기채, 금리 인하 시) | No | **장기채** (확신 있을 때만) |

### 10.3 금리 전망별 채권 선택 매트릭스

| 금리 전망 | 확신도 | 장기국공채 | 중장기채 | 단기채 | 예금 |
|----------|:------:|:----------:|:-------:|:------:|:----:|
| **금리 인하 확실** | 높음 | ✅ 추천 | ✅ 적합 | ⚠️ 기회비용 | ❌ |
| **금리 인하 예상** | 중간 | ⚠️ 일부 | ✅ 추천 | ✅ 적합 | ⚠️ |
| **금리 방향 불확실** | 낮음 | ❌ 회피 | ⚠️ 일부 | ✅ 추천 | ✅ 추천 |
| **금리 인상/유지** | 높음 | ❌ 손실 | ❌ 손실 | ✅ 추천 | ✅ 추천 |

### 10.4 안전자산 선택 의사결정 트리

```
                    [안전자산 30% 배분]
                           │
                           ▼
              ┌─ 금리 인하 전망이 확실한가? ─┐
              │                              │
             YES                            NO
              │                              │
              ▼                              ▼
    [장기/중장기 국공채]          ┌─ 금리 방향 불확실? ─┐
     • 금리 하락 시 자본이득         │                    │
     • 듀레이션 리스크 감수         YES                  NO (인상 전망)
                                    │                    │

                         • 금리 민감도 최소화       • 금리 상승 방어
```

### 10.5 예금 vs 채권형 펀드 비교 프로세스 (필수)

> ⚠️ **안전자산 배분 시 반드시 이 비교를 수행해야 합니다.**

#### 필수 비교 단계

```
Step 1: 현재 예금 금리 확인
   └─ funds/deposit_rates.json 참조
   └─ 퇴직연금 예금 금리 확인 (예: 3.60%)

Step 2: 채권형 펀드 실질 수익률 계산
   └─ 실질 수익률 = 1년 수익률 - 총보수
   └─ 예: 4.50% - 0.30% = 4.20%

Step 3: 비교 판단
   └─ IF 실질 수익률 > 예금 금리 + 0.5%p:
         THEN 채권형 펀드 선택 가능
      ELSE:
         THEN 예금 우선 추천
```

#### 비교 테이블 예시 (필수 포함)

| 구분 | 예금 | 채권형 펀드 |
|------|:----:|:----------:|
| **명목 수익률** | 3.60% | 4.50% |
| **총보수** | 0% | 0.30% |
| **실질 수익률** | **3.60%** | **4.20%** |
| **원금 보장** | ✅ | ❌ |
| **판단 기준** | - | 4.10% (3.60+0.5) |
| **결론** | 기준 | 선택 가능 (4.20 > 4.10) |

#### 판단 기준 상세

| 상황 | 조건 | 결정 | 근거 |
|------|------|------|------|
| **예금 선택** | 실질 수익률 ≤ 예금 + 0.5%p | 예금 | 원금 보장 가치 |
| **채권형 선택** | 실질 수익률 > 예금 + 0.5%p | 채권형 | 충분한 초과 수익 |
| **단기채 vs 장기채** | 금리 불확실 | 단기채 | 금리 리스크 최소화 |
| **크레딧 vs 국공채** | 안정성 우선 | 국공채 | 신용 리스크 없음 |

#### 데이터 확인 코드

```
1. 예금 금리 확인:
   Read funds/deposit_rates.json
   → rates.retirementDeposit.value (예: 3.60%)

2. 채권형 펀드 수익률 확인:
   Read funds/fund_data.json
   → 채권형 펀드의 return1y (예: 4.50%)

3. 채권형 펀드 총보수 확인:
   Read funds/fund_fees.json
   → 해당 펀드의 totalFee (예: 0.30%)

4. 실질 수익률 계산:
   실질 수익률 = return1y - totalFee
   = 4.50% - 0.30% = 4.20%

5. 비교 판단:
   판단 기준 = 예금 금리 + 0.5%p = 3.60% + 0.50% = 4.10%
   4.20% > 4.10% → 채권형 펀드 선택 가능
```

#### 비교 체크리스트

- [ ] deposit_rates.json에서 퇴직연금 예금 금리 확인
- [ ] 채권형 펀드 1년 수익률 확인 (fund_data.json)
- [ ] 채권형 펀드 총보수 확인 (fund_fees.json)
- [ ] 실질 수익률 계산 (수익률 - 총보수)
- [ ] 판단 기준 계산 (예금 금리 + 0.5%p)
- [ ] 비교 테이블 작성 (포트폴리오 추천 시 필수)
- [ ] 결론 명시 (예금/채권형 선택 및 근거)

- [ ] **투자 원칙 리마인더** (필수)
- [ ] 면책조항 포함

### 13.3 분석 원칙

- [ ] 데이터 기반 객관적 분석

- [ ] **안전자산은 예금 먼저 검토**

### 13.4 인덱스 vs 액티브 체크 (⚠️ 새 항목)

### 15.2 에이전트 역할

| 에이전트 | 역할 | 파일 |
|----------|------|------|
| `portfolio-coordinator` | 오케스트레이터, 전체 워크플로우 조율 | `.claude/agents/portfolio-coordinator.md` |
| `macro-outlook` | 거시경제 분석, 시장 전망 (신규) | `.claude/agents/macro-outlook.md` |

### 15.3 워크플로우 시퀀스

```
1. User → Coordinator: "공격형 포트폴리오 추천해줘"

2. Coordinator → macro-outlook: "거시경제 분석 및 시장 전망" (신규 Step 0)
   
3. macro-outlook → Coordinator: {
     recommendations: {
       riskWeight: 70,
       hedgeStrategy: "환노출",
       sectors: ["반도체", "AI", "로봇"],
       regions: { us: 60, kr: 20, em: 20 }
     },
     report: "00-macro-outlook.md"
   }

4. Coordinator → fund-portfolio: "fund_data.json + macro-outlook 기반 공격형 분석"
   
5. fund-portfolio → Coordinator: {
     recommendations: [...],
     analysis: "...",
     macroOutlookReflection: { ... },  // macro-outlook 반영 현황
     sources: [...]
   }

6. Coordinator → compliance-checker: "이 포트폴리오 검증해줘"
   
7. compliance-checker → Coordinator: {
     compliance: "pass/fail",
     violations: [...],
     corrective_actions: [...]
   }

8. IF compliance.fail:
   Coordinator → fund-portfolio: "규제 위반, 수정 필요"
   → Step 6 반복 (최대 3회)

9. Coordinator → output-critic: "최종 출력 검증"

10. output-critic → Coordinator: {
      verified: true/false,
      confidence_score: 0-100,
      issues: [...]
    }

11. Coordinator → User: [최종 포트폴리오 추천] (04-portfolio-summary.md)
```

### 15.5 Audit Trail 지원

포트폴리오 분석 시 `funds/scripts/audit_logger.js`를 사용하여 감사 로그를 생성할 수 있습니다:

```bash
# 로그 목록 조회
node funds/scripts/audit_logger.js list

# 예시 실행
node funds/scripts/audit_logger.js example
```

로그 기록 항목:
- 데이터 접근 (fund_data.json, fund_fees.json 등)
- 웹검색 쿼리 및 결과
- 에이전트 호출 및 응답
- 규제 준수 검증 결과
- 최종 추천 결과

### 15.6 단독 사용 vs Multi-Agent 사용

| 모드 | 사용 시기 | 장점 | 단점 |
|------|----------|------|------|
| **단독** | 빠른 분석, 간단한 요청 | 빠른 응답 | 교차 검증 없음 |
| **Multi-Agent** | 복잡한 분석, 프로덕션 | 규제 준수 보장, 환각 방지 | 응답 시간 증가 |

**권장**: 
- 단순 펀드 조회: 단독 사용
- 포트폴리오 추천: Multi-Agent 사용
