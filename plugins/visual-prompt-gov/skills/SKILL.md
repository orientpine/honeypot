# 슬라이드 프롬프트 생성기

연구 계획서 문서를 분석하여 핵심 개념을 추출하고, 각 개념에 적합한 시각화 레이아웃으로 이미지 생성 프롬프트를 작성합니다.

## 사용자 입력 스키마

### 필수 입력

| 항목 | 설명 | 예시 |
|------|------|------|
| 입력 문서 | 분석할 연구 계획서/사업 계획서 경로 | `연구계획서.md`, `사업계획서.pdf` |

### 선택 입력

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 색상 테마 | 5가지 테마 중 선택 | `gov-blue` (정부 블루) |
| 출력 폴더 | 프롬프트 저장 위치 | `slide_image_gen/prompts/` |
| 슬라이드 수 | 생성할 슬라이드 개수 | 문서에서 자동 추출 |

---

## 워크플로우

### Phase 1: 문서 분석

```
+-- Step 1-1. 입력 문서 읽기
|   +-- 연구 계획서, 사업 계획서, 기술 문서 등
|   +-- 문서 구조 파악 (목차, 섹션)
|
+-- Step 1-2. 핵심 개념 추출
|   +-- 비전/목표
|   +-- 세부 기술/기능
|   +-- 핵심 메커니즘
|   +-- 단계별 계획
|   +-- 추진체계
|   +-- 협력 관계
|
+-- Step 1-3. 슬라이드 구성 계획
    +-- 추출된 개념별 슬라이드 제목 결정
    +-- 각 슬라이드에 적합한 레이아웃 유형 선택
    +-- 슬라이드 순서 결정
```

### Phase 2: 테마 선택

```
+-- Step 2-1. 분야 키워드 분석
|   +-- 문서에서 분야 키워드 추출
|   +-- 정책/공공 → gov-blue
|   +-- 기술/AI → tech-green
|   +-- 제조/건설 → industry-gray
|   +-- 바이오/의료 → research-purple
|   +-- 환경/에너지 → eco-mint
|
+-- Step 2-2. 테마 자동 추천 또는 사용자 선택
|   +-- 키워드 기반 자동 추천
|   +-- 사용자 오버라이드 가능
|
+-- Step 2-3. 테마 색상 로드
    +-- references/prompt_style_guide.md에서 해당 테마 색상 로드
    +-- 메인, 서브, 강조, 배경 4색 확정
```

**색상 테마 옵션:**

| 테마 ID | 테마명 | 적합 분야 | 메인 색상 |
|:---|:---|:---|:---|
| `gov-blue` | 정부 블루 | 정책, 공공, R&D | #1E3A5F |
| `tech-green` | 테크 그린 | 기술, AI, 혁신 | #1B4332 |
| `industry-gray` | 산업 그레이 | 제조, 건설, 인프라 | #2C3E50 |
| `research-purple` | 연구 퍼플 | 바이오, 의료, 기초과학 | #4A1A6B |
| `eco-mint` | 환경 민트 | 환경, 에너지, 지속가능 | #0B525B |

### Phase 3: 레이아웃 선택

[references/layout_types.md](references/layout_types.md) 참조하여 개념에 맞는 레이아웃 선택:

| 개념 유형 | 레이아웃 |
|:---|:---|
| 비전, 목표 계층 | 계층형 피라미드 |
| 통합 플랫폼, 범용 시스템 | 중앙 방사형 |
| 데이터 파이프라인, 순환 프로세스 | 원형 순환 |
| 하드웨어 모듈, 구성품 | 3D 분해도 |
| 실제 vs 가상, 전후 비교 | 좌우 병렬 비교 |
| 연도별 계획, 로드맵 | 수평 타임라인 |
| 조직 구조, 협력 관계 | 조직도 + 네트워크 |
| 분류 체계, 매트릭스 | 표 구조 |
| 아키텍처 레이어, 계층적 포함 | 동심원 레이어 |
| 다중 주체 협업, 워크플로우 | Swimlane 프로세스 |
| 우선순위, 포지셔닝, 전략 맵 | 매트릭스 그리드 |
| 선별 과정, 전환 파이프라인 | Funnel/깔때기 |

### Phase 4: 프롬프트 작성

[references/prompt_style_guide.md](references/prompt_style_guide.md)의 스타일 가이드 적용.
[assets/output_template/prompt_template.md](assets/output_template/prompt_template.md) 템플릿 기반으로 작성.

```
+-- Step 4-1. 프롬프트 초안 작성
|   +-- 메타 섹션 (1~6, 11): 영어로 작성
|   +-- 콘텐츠 섹션 (7~10, 12): 한글로 작성
|   +-- 선택된 테마 색상 적용
|
+-- Step 4-2. 텍스트 규격 검증
|   +-- 모든 텍스트 최소 14pt 이상
|   +-- 대제목 28pt 이상
|   +-- 슬라이드 전체 텍스트 요소 25개 이하
|   +-- 한 박스 내 텍스트 3줄 이하
|
+-- Step 4-3. 품질 체크리스트 확인
    +-- references/prompt_style_guide.md 하단 체크리스트 적용
```

**프롬프트 구조:**

```
[INSTRUCTION - DO NOT RENDER IN IMAGE]

1. Purpose (영어)
2. Tone & Manner (영어)
3. Style (영어)
4. Colors (영어 - 선택된 테마 색상)
5. Lighting (영어)
6. Resolution (영어)

[슬라이드 콘텐츠 - 한글]

7. 슬라이드 레이아웃
8. 상단 타이틀
9. [메인 콘텐츠 섹션들]
10. 인포그래픽 디테일
11. Forbidden Elements (영어)
12. 최종 결과물 목표
```

### Phase 5: 출력

```
+-- Step 5-1. 출력 폴더 생성
|   +-- slide_image_gen/prompts/ 폴더 생성 (없으면)
|
+-- Step 5-2. 프롬프트 파일 저장
|   +-- 파일명: [번호]_[슬라이드제목].md
|   +-- 인코딩: UTF-8
|
+-- Step 5-3. 인덱스 파일 생성
    +-- prompt_index.md 생성
    +-- 전체 슬라이드 목록 및 레이아웃 정보 기록
```

## 텍스트 규격 (필수)

**14pt 미만 텍스트는 이미지 렌더링 시 깨짐 발생. 반드시 준수.**

| 요소 | 최소 크기 | 권장 크기 | 최대 글자수 |
|:---|:---:|:---:|:---:|
| 대제목 | **28pt** | 32-36pt | 20자 |
| 섹션 제목 | **20pt** | 22-24pt | 30자 |
| 박스 라벨 | **14pt** | 16-18pt | 15자 |
| 소항목 | **14pt** | 14-16pt | 25자 |
| 캡션/주석 | **사용 금지** | - | - |

**텍스트 밀도 제한:**
- 한 박스 내 텍스트: 최대 3줄
- 슬라이드 전체 텍스트 요소: 최대 25개
- 라벨 간 최소 간격: 20px

---

## 공통 규칙

- **텍스트**: 한글 기본, 기술 용어는 영어 병기
- **스타일**: 정돈된 3D 렌더링 + 미니멀 인포그래픽
- **시점**: 약한 Isometric
- **금지**: 만화풍, SF/사이버펑크, 어두운 배경, 12pt 이하 텍스트
- **해상도**: 8K Quality
