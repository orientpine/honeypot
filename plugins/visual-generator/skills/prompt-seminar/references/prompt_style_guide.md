# 세미나 프롬프트 스타일 가이드

## 프롬프트 구조

모든 세미나 프롬프트는 다음 4개 블록으로 구성:

```
[INSTRUCTION BLOCK - DO NOT RENDER]
1. Purpose (영어 작성)
2. Visual Concept (영어 작성 - 시각적 메타포)
3. Style (영어 작성)
4. Colors (영어 작성)
5. Lighting & Mood (영어 작성)
6. Resolution (영어 작성)

[CONFIGURATION BLOCK - DO NOT RENDER]
- Text Specifications (영어 작성)
- Visual Specifications (영어 작성)
- Layout Combination (영어 작성 - 복합 레이아웃인 경우)

[CONTENT BLOCK - FOR IMAGE RENDERING]
7. 핵심 메시지 (선택한 단일 언어)
8. 도메인 키워드 (선택한 단일 언어)
9. 시각 구성 (선택한 단일 언어)
10. 텍스트 요소 (선택한 단일 언어)

[FORBIDDEN ELEMENTS - DO NOT RENDER]
11. Forbidden Elements (영어 작성)
12. 최종 결과물 목표 (선택한 단일 언어)
```

## 메타 섹션 작성 규칙 (중요)

**Gemini API 렌더링 문제 방지를 위해 메타/설정 섹션은 영어로 작성:**

```
==================================================
[INSTRUCTION BLOCK - DO NOT RENDER IN IMAGE]
==================================================

Purpose:
Generate a seminar presentation visualization for internal workshops and knowledge sharing.
[개념의 목적을 영어로 1-2문장 기술]

Visual Concept:
[시각적 메타포를 영어로 상세 기술]
- Central metaphor: [핵심 비유]
- Visual elements: [시각 요소들]
- Spatial arrangement: [공간 배치]
- Layout combination (if applicable): [복합 레이아웃 조합]

Style:
- Professional 3D rendering with rich infographic elements
- Seminar presentation aesthetic (informative, engaging)
- Sans-serif typography, clear hierarchy
- Balance between visual impact and information density
```

**콘텐츠 섹션은 단일 언어 원칙:**
- 한글 OR 영어 중 하나만 선택
- 언어 병기 절대 금지 (한글(English) 형식 금지)
- 전문 용어도 선택한 언어로 통일

## 세미나 발표 스타일 핵심 원칙

### 1. Rich Information with Clarity

```
풍부한 정보와 명확성의 균형:
- 한 이미지에 1-3개의 핵심 개념
- 계층적 정보 구조 (대제목 > 중제목 > 소제목 > 라벨)
- 정보 밀도 높지만 가독성 유지
- 5초 안에 핵심 파악, 30초 안에 세부 이해 가능
```

### 2. Professional Presentation Style

```
전문적 발표 스타일:
- 세미나/워크숍 맥락에 적합한 표현
- 청중의 관심과 이해를 동시에 확보
- 시각적 임팩트와 정보 전달의 조화
- 사내 발표 자료 품질 기준
```

### 3. Visual Impact with Density

```
시각적 임팩트와 정보 밀도:
- 텍스트 요소 20-30개 범위
- 복합 레이아웃으로 다층적 정보 전달
- 의도적 색상 대비로 주목도 확보
- 3D 렌더링 + 아이콘 하이브리드
```

### 4. Single Language Consistency

```
단일 언어 일관성:
- 한글 OR 영어 중 하나만 선택
- 언어 병기 절대 금지
- 전문 용어도 선택한 언어로 통일
- INSTRUCTION 블록에만 번역 힌트 포함 가능
```

### 테마 선택 가이드

개념의 성격에 따라 9가지 테마 중 선택:

| 테마 ID | 테마명 | 적합 개념 |
|:---|:---|:---|
| `technical-report` | 기술 보고서 | 국책과제, 연구논문, 학술발표 |
| `clarity` | 명료 | 설명, 정의, 구조, 분류 |
| `tech-focus` | 테크 | 기술, AI, 데이터, 시스템 |
| `growth` | 성장 | 발전, 학습, 진화, 향상 |
| `connection` | 연결 | 네트워크, 관계, 협력, 소통 |
| `innovation` | 혁신 | 변화, 창조, 돌파, 전환 |
| `knowledge` | 지식 **(신규)** | 큐레이션, 학습, 정보 관리 **(권장)** |

## 시각 스타일 가이드

### 3D 렌더링 + 아이콘 하이브리드 (세미나용)

```
Style:
- Professional 3D rendering with rich infographic elements
- Seminar presentation aesthetic (informative, engaging)
- Balanced lighting with clear shadows
- Mix of matte and subtle glossy surfaces
- Isometric or slight perspective view (15-30 degrees)
- Sans-serif typography, clear hierarchy
- Combination of 3D objects and flat icons
```

**적합한 개념:**
- 플랫폼 아키텍처
- 시스템 구성
- 프로세스 플로우
- 정보 구조

**3D 사용 케이스:**
- 물리적 구조 (하드웨어, 인프라)
- 계층적 아키텍처
- 공간적 배치

**아이콘 사용 케이스:**
- 추상적 개념 (알고리즘, 프로세스)
- 데이터 흐름
- 관계/연결

## 텍스트 규격

**세미나 발표 원칙: 풍부한 정보와 가독성의 균형**

> ⚠️ **중요**: 아래 크기 규격은 내부 참고용입니다. **프롬프트에 "pt", "px" 단위를 포함하지 마세요.**

| 요소 | 크기 등급 | 최대 글자수 | 프롬프트 표기 |
|:---|:---:|:---:|:---|
| 대제목 | 대형 (XL) | 15자 | "대형" 또는 "prominent" |
| 중제목 | 중형 (L) | 20자 | "중형" 또는 "medium" |
| 소제목 | 소형 (M) | 15자 | "소형" 또는 "small" |
| 라벨 | 최소형 (S) | 10자 | "최소형" 또는 "minimal" |
| 캡션/주석 | **사용 금지** | - | 프롬프트에 포함 금지 |

**텍스트 밀도 제한:**
- 이미지 전체 텍스트 요소: **20-30개 (최대 30개)**
- 한 영역 내 라벨: **최대 6개**
- 문장 사용 가능 (간결한 경우), 키워드 우선
- **폰트 크기 숫자 (14pt, 32px 등) 프롬프트 포함 절대 금지**

```markdown

```

### 영어 선택 시

```markdown
### Text Elements
| Element | Content | Position |
|---------|---------|----------|
| Main Title | Knowledge Curation Platform | Center Top |

| Domain Labels | Research / Education / Industry | Each Section Top |
```

### 금지 패턴

```markdown

|------|------|------|
| 메인 타이틀 | 지식 큐레이션 플랫폼 (Knowledge Curation Platform) | 중앙 상단 |

✅ 올바른 예시 (단일 언어):
| 요소 | 내용 | 위치 |
|------|------|------|

```

## 금지 요소 목록

프롬프트에 다음 요소를 포함하지 마세요:

### 렌더링 방지

- ⚠️ **폰트 크기 숫자** (14pt, 32pt, 48px 등)
- ⚠️ **"pt", "px" 단위**가 포함된 모든 텍스트
- 테이블의 "크기" 열 (내용과 위치만 기술)

### 언어 병기 방지

- ⚠️ **한글(English) 형식** 절대 금지
- ⚠️ **영어 (한글) 형식** 절대 금지
- 괄호 안 번역 표기 모두 금지

### 스타일 금지

- 만화풍, 캐릭터
- SF/사이버펑크 스타일
- 어두운 배경
- 과도한 장식 요소
- 중국어(汉字), 일본어(仮名) 등 비한글 아시아 문자
