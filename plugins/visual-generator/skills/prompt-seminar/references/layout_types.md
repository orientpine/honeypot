# 세미나 시각화 유형

개념의 성격에 따라 적절한 시각적 메타포를 선택합니다.
각 유형은 핵심 원칙, 시각 구성, 적합/부적합 케이스를 포함합니다.

**핵심 원칙**: 모든 유형에서 텍스트는 20-30개 범위로 유지하고, 시각적 메타포가 개념을 전달해야 합니다.

---

## 1. 플로우 메타포 (Flow Metaphor)

**적합한 개념**: 과정, 변환, 단계, 파이프라인, 워크플로우

### 핵심 아이디어

```
"무언가가 A에서 B로 변화/이동한다"

입력 → [변환] → 출력
시작 → 과정 → 결과
원료 → 가공 → 제품
```

### 시각 구성

```
     ┌─────┐        ┌─────┐        ┌─────┐
     │  A  │───────▶│  B  │───────▶│  C  │
     └─────┘        └─────┘        └─────┘
       ↑              ↑              ↑
     시작           변환           결과
```

### 시각화 원칙

- **방향성**: 좌→우 또는 상→하 일관된 흐름
- **연결**: 화살표로 명확한 이동 표현
- **단계 수**: 3-5단계 권장
- **중간 강조**: 핵심 변환점에 시각적 강조

### 스타일 옵션

**3D 스타일**: 파이프/컨베이어 벨트, 물 흐름
**아이콘 스타일**: 깔끔한 도형과 화살표

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 3-5개 |
| 라벨 글자수 | 10자 이내 |
| 텍스트 요소 | 15-20개 |

### 적합/부적합 케이스

✅ 데이터 처리 파이프라인, 머신러닝 학습 과정
❌ 순환 프로세스 (→ 순환 메타포)

---

## 2. 구조 메타포 (Structure Metaphor)

**적합한 개념**: 계층, 구성, 아키텍처, 레이어, 스택

### 핵심 아이디어

```
"여러 층/부분이 쌓여서 전체를 이룬다"

    [상위]
   --------
    [중간]
   --------
    [기반]
```

### 시각 구성

```
         ┌─────────────────┐
         │    응용 계층     │
         ├─────────────────┤
         │    비즈니스     │
         ├─────────────────┤
         │    데이터       │
         └─────────────────┘
```

### 시각화 원칙

- **계층성**: 상하 또는 내외 구조
- **포함 관계**: 하위가 상위를 지탱
- **레이어 수**: 3-4층 권장

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 레이어 수 | 3-4개 |
| 텍스트 요소 | 18-25개 |

✅ 소프트웨어 아키텍처, 조직 구조
❌ 동등한 관계 (→ 네트워크 메타포)

---

## 3. 네트워크 메타포 (Network Metaphor)

**적합한 개념**: 관계, 연결, 상호작용, 생태계

### 핵심 아이디어

```
"여러 요소가 서로 연결되어 있다"

  A ←→ B
  ↕    ↕
  C ←→ D
```

### 시각 구성

```
       ┌───┐
    ┌──│ A │──┐
    │  └───┘  │
  ┌─▼─┐    ┌─▼─┐
  │ B │────│ C │
  └───┘    └───┘
```

### 시각화 원칙

- **노드**: 핵심 요소 명확히 표현
- **엣지**: 관계의 방향성/강도
- **밀도**: 과도한 연결 지양

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 노드 수 | 4-8개 |
| 텍스트 요소 | 20-28개 |

✅ 협업 관계, 데이터 흐름
❌ 단방향 흐름 (→ 플로우 메타포)

---

## 4. 대비 메타포 (Contrast Metaphor)

**적합한 개념**: 비교, 차이, 전후, Before/After

### 핵심 아이디어

```
"A와 B는 이렇게 다르다"

  [A]  vs  [B]
 기존      개선
```

### 시각 구성

```
  ┌─────────┐    ┌─────────┐
  │  Before │    │  After  │
  │         │    │         │
  │  문제점  │    │  해결책  │
  └─────────┘    └─────────┘
```

### 시각화 원칙

- **대칭성**: 좌우 또는 상하 균형
- **차이 강조**: 색상/크기로 대비
- **비교 항목**: 3-5개 항목 비교

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 비교 항목 | 3-5개 |
| 텍스트 요소 | 18-24개 |

✅ 기술 비교, 개선 효과
❌ 단일 개념 (→ 중심 메타포)

---

## 5. 진화 메타포 (Evolution Metaphor)

**적합한 개념**: 성장, 발전, 단계적 향상

### 핵심 아이디어

```
"시간에 따라 점진적으로 발전한다"

 V1.0 → V2.0 → V3.0
  ↑      ↑      ↑
 기본   개선   완성
```

### 시각 구성

```
  ┌───┐   ┌────┐   ┌─────┐
  │ 1 │──▶│ 2  │──▶│  3  │
  └───┘   └────┘   └─────┘
  작음     중간      큼
```

### 시각화 원칙

- **크기 변화**: 점진적 증가
- **방향성**: 좌→우 또는 하→상
- **단계 표시**: 명확한 버전/단계

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 3-4개 |
| 텍스트 요소 | 15-22개 |

✅ 제품 로드맵, 기술 발전
❌ 순환 과정 (→ 순환 메타포)

---

## 6. 중심 메타포 (Central Metaphor)

**적합한 개념**: 핵심, 허브, 중앙 집중

### 핵심 아이디어

```
"중심에서 모든 것이 시작/연결된다"

      ┌─ A
  [중심]─ B
      └─ C
```

### 시각 구성

```
        ┌───┐
     ┌──│ A │──┐
     │  └───┘  │
   ┌─▼─┐    ┌─▼─┐
   │ B │    │ C │
   └───┘    └───┘
```

### 시각화 원칙

- **중심 강조**: 크기/색상으로 강조
- **방사형**: 균등한 배치
- **연결**: 중심에서 외곽으로

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 주변 요소 | 4-8개 |
| 텍스트 요소 | 18-25개 |

✅ 플랫폼 아키텍처, 허브 시스템
❌ 동등한 관계 (→ 네트워크 메타포)

---

## 7. 순환 메타포 (Cycle Metaphor)

**적합한 개념**: 반복, 순환, 피드백 루프

### 핵심 아이디어

```
"끝이 다시 시작으로 이어진다"

  A → B → C
  ↑       ↓
  └───────┘
```

### 시각 구성

```
      ┌───┐
   ┌──│ A │──┐
   │  └───┘  │
 ┌─▼─┐    ┌─▼─┐
 │ D │    │ B │
 └─┬─┘    └─┬─┘
   │  ┌───┐ │
   └──│ C │─┘
      └───┘
```

### 시각화 원칙

- **원형 배치**: 시계 방향 흐름
- **연속성**: 끝과 시작 연결
- **단계 수**: 4-6단계

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 4-6개 |
| 텍스트 요소 | 20-28개 |

✅ PDCA 사이클, 반복 프로세스
❌ 일방향 흐름 (→ 플로우 메타포)

---

## 8. 그룹 메타포 (Group Metaphor)

**적합한 개념**: 분류, 카테고리, 종류

### 핵심 아이디어

```
"여러 그룹으로 나뉜다"

 [그룹A]  [그룹B]  [그룹C]
   ↓        ↓        ↓
 항목들   항목들   항목들
```

### 시각 구성

```
 ┌─────────┐ ┌─────────┐ ┌─────────┐
 │ 그룹 A  │ │ 그룹 B  │ │ 그룹 C  │
 ├─────────┤ ├─────────┤ ├─────────┤
 │ 항목1   │ │ 항목1   │ │ 항목1   │
 │ 항목2   │ │ 항목2   │ │ 항목2   │
 └─────────┘ └─────────┘ └─────────┘
```

### 시각화 원칙

- **명확한 구분**: 그룹 간 경계
- **균등 배치**: 동등한 중요도
- **그룹 수**: 3-4개 권장

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 그룹 수 | 3-4개 |
| 텍스트 요소 | 22-30개 |

✅ 제품 분류, 사용자 세그먼트
❌ 계층 구조 (→ 구조 메타포)

---

## 9. 동심원 메타포 (Concentric Metaphor)

**적합한 개념**: 레이어, 포함, 범위

### 핵심 아이디어

```
"안쪽에서 바깥쪽으로 확장된다"

   ┌─────────┐
   │ ┌─────┐ │
   │ │ 핵심 │ │
   │ └─────┘ │
   └─────────┘
```

### 시각 구성

```
        ┌───────────────┐
        │   외부 계층    │
        │ ┌───────────┐ │
        │ │ 중간 계층  │ │
        │ │ ┌───────┐ │ │
        │ │ │ 핵심  │ │ │
        │ │ └───────┘ │ │
        │ └───────────┘ │
        └───────────────┘
```

### 시각화 원칙

- **중심에서 외곽**: 확장 방향
- **레이어 수**: 3-4층
- **포함 관계**: 명확한 계층

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 레이어 수 | 3-4개 |
| 텍스트 요소 | 18-24개 |

✅ 영향 범위, 보안 계층
❌ 독립적 요소 (→ 그룹 메타포)

## 10. Swimlane 메타포 (Swimlane Metaphor)

**적합한 개념**: 다중 주체, 협업, 역할 분담

### 핵심 아이디어

```
"여러 주체가 동시에 각자의 역할을 수행한다"

 [주체A] ─────────────────▶
 [주체B] ─────────────────▶
 [주체C] ─────────────────▶
```

### 시각 구성

```
 ┌─────────┬───────┬───────┬───────┐
 │ 주체 A  │ 작업1 │ 작업2 │ 작업3 │
 ├─────────┼───────┼───────┼───────┤
 │ 주체 B  │ 작업1 │ 작업2 │ 작업3 │
 ├─────────┼───────┼───────┼───────┤
 │ 주체 C  │ 작업1 │ 작업2 │ 작업3 │
 └─────────┴───────┴───────┴───────┘
```

### 시각화 원칙

- **수평 레인**: 주체별 구분
- **시간 흐름**: 좌→우
- **상호작용**: 레인 간 화살표

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 레인 수 | 3-4개 |
| 텍스트 요소 | 24-30개 |

✅ 협업 워크플로우, 다중 주체 프로세스
❌ 단일 주체 (→ 플로우 메타포)

---

## 11. 전략맵 메타포 (Strategy Map Metaphor)

**적합한 개념**: 포지셔닝, 우선순위, 매트릭스

### 핵심 아이디어

```
"2개 축으로 요소를 배치한다"

  높음 │  B    D
       │
  낮음 │  A    C
       └─────────
       낮음  높음
```

### 시각 구성

```
       │
   높음 │  ┌───┐    ┌───┐
       │  │ B │    │ D │
       │  └───┘    └───┘
   ────┼──────────────────
       │  ┌───┐    ┌───┐
   낮음 │  │ A │    │ C │
       │  └───┘    └───┘
       └─────────────────
         낮음      높음
```

### 시각화 원칙

- **2축 매트릭스**: X축, Y축 명확
- **사분면**: 4개 영역 구분
- **요소 배치**: 위치가 의미 전달

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 요소 수 | 6-12개 |
| 텍스트 요소 | 20-28개 |

✅ 우선순위 매트릭스, 포트폴리오 분석
❌ 단순 분류 (→ 그룹 메타포)

## 12. 깔때기 메타포 (Funnel Metaphor)

**적합한 개념**: 선별, 축소, 전환 과정

### 핵심 아이디어

```
"많은 것이 점점 줄어든다"

  ┌─────────┐
  │  많음   │
  └────┬────┘
       │
    ┌──▼──┐
    │ 중간 │
    └──┬──┘
       │
     ┌─▼─┐
     │적음│
     └───┘
```

### 시각 구성

```
  ┌───────────────┐
  │   1000명      │
  └───────┬───────┘
          │
      ┌───▼───┐
      │ 100명 │
      └───┬───┘
          │
        ┌─▼─┐
        │10명│
        └───┘
```

### 시각화 원칙

- **상→하 축소**: 점진적 감소
- **단계 표시**: 각 단계 수치
- **전환율**: 단계 간 비율

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 3-5개 |
| 텍스트 요소 | 15-22개 |

✅ 세일즈 퍼널, 사용자 전환
❌ 확장 과정 (→ 진화 메타포)

## 13. Hub-Network 메타포 (Hub-Network Metaphor) - 신규

**적합한 개념**: 중앙 허브 + 방사형 노드 + 작은 연결 노드

### 핵심 아이디어

```
"중앙 허브에서 주요 노드로 연결되고, 각 주요 노드에 작은 노드들이 연결된다"

        ┌─ a1
    ┌─ A ─ a2
    │   └─ a3
 [허브]
    │   ┌─ b1
    └─ B ─ b2
        └─ b3
```

### 시각 구성

```
          ┌───┐
       ┌──│a1 │
     ┌─▼─┐└───┘
  ┌──│ A │──┐
  │  └───┘  │┌───┐
  │         ││a2 │
┌─▼──┐      │└───┘
│허브│      │
└─┬──┘      │┌───┐
  │         ││a3 │
  │  ┌───┐  │└───┘
  └──│ B │──┘
     └─┬─┘
       │  ┌───┐
       └──│b1 │
          └───┘
```

### 시각화 원칙

- **3단계 계층**: 허브 → 주요 노드 → 작은 노드
- **방사형 배치**: 허브 중심 균등 배치
- **연결 강도**: 허브-주요 노드는 굵게, 주요-작은 노드는 얇게

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 주요 노드 | 4-6개 |
| 작은 노드 | 8-15개 |
| 텍스트 요소 | 25-30개 |

✅ 지식 큐레이션 플랫폼, 콘텐츠 분류 시스템
❌ 단순 중심 구조 (→ 중심 메타포)

---

## 14. Section-Flow 메타포 (Section-Flow Metaphor) - 신규

**적합한 개념**: 상단 메인 비주얼 + 하단 3단계 플로우

### 핵심 아이디어

```
"상단에 큰 메인 개념, 하단에 세부 단계"

 ┌─────────────────────┐
 │    메인 비주얼       │
 └─────────────────────┘
  ┌─────┐ ┌─────┐ ┌─────┐
  │단계1│→│단계2│→│단계3│
  └─────┘ └─────┘ └─────┘
```

### 시각 구성

```
 ┌───────────────────────────┐
 │                           │
 │      메인 개념 비주얼      │
 │                           │
 └───────────────────────────┘
         ↓
 ┌─────┐    ┌─────┐    ┌─────┐
 │단계1│───▶│단계2│───▶│단계3│
 └─────┘    └─────┘    └─────┘
```

### 시각화 원칙

- **상하 분할**: 상단 60%, 하단 40%
- **메인 강조**: 상단 큰 비주얼
- **플로우 연결**: 하단 단계 간 화살표

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 메인 요소 | 1개 |
| 하단 단계 | 3-4개 |
| 텍스트 요소 | 20-26개 |

✅ 플랫폼 소개 + 사용 단계, 시스템 개요 + 워크플로우
❌ 동등한 요소 (→ 그룹 메타포)

## 15. Card-Grid 메타포 (Card-Grid Metaphor) - 신규

**적합한 개념**: 동일 크기 카드 그리드 + 우측/하단 태그

### 핵심 아이디어

```
"여러 카드가 그리드로 배치되고, 각 카드에 태그가 붙는다"

 ┌─────┐ ┌─────┐ ┌─────┐
 │카드1│ │카드2│ │카드3│
 │#태그│ │#태그│ │#태그│
 └─────┘ └─────┘ └─────┘
 ┌─────┐ ┌─────┐ ┌─────┐
 │카드4│ │카드5│ │카드6│
 │#태그│ │#태그│ │#태그│
 └─────┘ └─────┘ └─────┘
```

### 시각 구성

```
 ┌───────────┐ ┌───────────┐ ┌───────────┐
 │  카드 1   │ │  카드 2   │ │  카드 3   │
 │           │ │           │ │           │
 │  #태그A   │ │  #태그B   │ │  #태그C   │
 └───────────┘ └───────────┘ └───────────┘
 ┌───────────┐ ┌───────────┐ ┌───────────┐
 │  카드 4   │ │  카드 5   │ │  카드 6   │
 │           │ │           │ │           │
 │  #태그D   │ │  #태그E   │ │  #태그F   │
 └───────────┘ └───────────┘ └───────────┘
```

### 시각화 원칙

- **균등 그리드**: 모든 카드 동일 크기
- **태그 배치**: 우측 하단 또는 하단 중앙
- **카드 수**: 6-9개 권장

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 카드 수 | 6-9개 |
| 태그 수 | 카드당 1-2개 |
| 텍스트 요소 | 24-30개 |

✅ 기능 소개, 제품 카탈로그, 사례 모음
❌ 계층 구조 (→ 구조 메타포)

## 복합 레이아웃 조합 가이드

세미나 발표 자료는 한 슬라이드에 여러 레이아웃을 조합할 수 있습니다.

### 조합 규칙

- **최대 3개** 메타포 조합 허용
- 각 메타포는 **명확히 구분된 영역** 배치
- 전체 구성의 **시각적 균형** 유지
- 색상 테마는 **전체 일관성** 유지

### 조합 예시

**예시 1: Hub-Network (중앙) + Card-Grid (하단)**
```
 ┌─────────────────────────┐
 │   Hub-Network 영역      │
 │   (중앙 허브 + 노드)     │
 └─────────────────────────┘
 ┌─────┐ ┌─────┐ ┌─────┐
 │카드1│ │카드2│ │카드3│
 └─────┘ └─────┘ └─────┘
```

**예시 2: Section-Flow (상단) + 네트워크 (중앙) + 표 (하단)**
```
 ┌─────────────────────────┐
 │   메인 비주얼 (30%)      │
 └─────────────────────────┘
 ┌─────────────────────────┐
 │   네트워크 (40%)         │
 └─────────────────────────┘
 ┌─────────────────────────┐
 │   비교 표 (30%)          │
 └─────────────────────────┘
```

**예시 3: 플로우 (좌측) + 구조 (우측)**
```
 ┌───────────┬───────────┐
 │           │           │
 │  플로우   │   구조    │
 │  (50%)    │   (50%)   │
 │           │           │
 └───────────┴───────────┘
```

### 조합 시 주의사항

- **주 메타포 1개 + 보조 메타포 1-2개** 구조 권장
- 각 메타포 영역 크기 비율: **주 60-70%, 보조 30-40%**
- 레이아웃 간 **시각적 연결성** 확보 (화살표, 선 등)
- 텍스트 밀도: 전체 합산 **20-30개** 유지
