# 세미나 프롬프트 생성기 (Visual Prompt Seminar)

사내 세미나/워크숍 발표 자료 스타일의 이미지 생성 프롬프트를 작성합니다.
**풍부한 정보 전달** 원칙으로 청중의 이해와 기억을 돕는 시각 자료를 만듭니다.

---

## prompt-concept와의 차이

| 항목 | prompt-concept | prompt-seminar |
|------|----------------|----------------|
| 스타일 | 기술 보고서/학술 발표 | 사내 세미나/워크숍 |
| 정보량 | 적정 (12-15개) | 풍부 (20-30개) |
| 시각화 | 직관적 개념 다이어그램 | 정보 밀도 높은 인포그래픽 |
| 레이아웃 | 단일 메타포 | 복합 레이아웃 (최대 3개 조합) |
| 텍스트 | 한글 (영어 힌트는 INSTRUCTION 블록) | 단일 언어 (한글 OR 영어, 병기 금지) |
| 테마 | 6개 | 9개 (기존 6개 + knowledge, presentation, workshop) |
| 목적 | 개념 이해 + 전문성 | 정보 전달 + 시각적 임팩트 |

## 사용자 입력 스키마

### 필수 입력

| 항목 | 설명 | 예시 |
|------|------|------|
| 핵심 개념 | 시각화할 개념/주제 | "Knowledge Curation 플랫폼", "AI 기반 추천 시스템" |

### 선택 입력

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 시각화 유형 | 선호하는 시각화 방식 (24종) | 자동 선택 |
| 무드 테마 | 9가지 무드 중 선택 | `presentation` |
| 도메인 키워드 | 보존해야 할 전문용어 목록 | - |
| 언어 선택 | 한글 또는 영어 (단일 언어) | 한글 |
| 출력 폴더 | 프롬프트 저장 위치 | `concept_image_gen/prompts/` |
| 슬라이드 수 | 생성할 이미지 개수 | 1 |

## 워크플로우

### Phase 1: 개념 분석

```
+-- Step 1-1. 핵심 개념 파악
|   +-- 사용자 입력에서 핵심 개념 추출
|   +-- 개념의 본질적 특성 식별
|   +-- 시각화 가능한 요소 도출
|
+-- Step 1-2. 도메인 키워드 식별
|   +-- 보존해야 할 전문용어 추출
|   +-- 일반화 금지 용어 목록 작성
|
+-- Step 1-3. 시각적 메타포 탐색
|   +-- 개념을 대변할 수 있는 시각적 비유 탐색
|   +-- 복합 레이아웃 필요성 판단 (단일 vs 조합)
|   +-- 정보 밀도 높은 구조 선택
|
+-- Step 1-4. 핵심 메시지 정의
|   +-- 전달할 주요 메시지 정의 (1-3개)
|   +-- 이미지를 본 후 기억할 핵심 내용
|   +-- 세부 정보 우선순위 결정
|
+-- Step 1-5. 언어 선택
    +-- 한글 또는 영어 중 단일 언어 선택
    +-- 병기 금지 원칙 확인
```

### Phase 2: 무드 테마 선택

```
+-- Step 2-1. 개념 성격 분석
|   +-- 국책과제/학술 → technical-report
|   +-- 기술적/과학적 → tech-focus
|   +-- 성장/발전 → growth
|   +-- 연결/관계 → connection
|   +-- 명확/단순 → clarity
|   +-- 혁신/변화 → innovation
|   +-- 지식/큐레이션 → knowledge (신규)
|   +-- 발표/프레젠테이션 → presentation (신규)
|   +-- 워크숍/협업 → workshop (신규)
|
+-- Step 2-2. 테마 색상 로드
```

#### 신규 레이아웃 활용 가이드 (9종)

**Gov 전용 레이아웃 (4종)**:
- **계층형 피라미드 (Pyramid)**: 비전→전략→실행의 위계/우선순위를 3-5단으로 정리 (전략 체계, 가치/우선순위 구조).
- **3D 분해도 (Exploded View)**: 코어+모듈 조립 관계를 분해해 구성요소/인터페이스를 직관적으로 설명 (플랫폼/장치/모듈 아키텍처).
- **수평 타임라인 (Horizontal Timeline)**: 연도/분기/단계별 마일스톤을 좌→우로 배치 (사업/과제 일정, 추진 로드맵).
- **조직-네트워크 (Org-Network)**: 조직 구조(위계)와 협력 관계(연결)를 동시에 표현 (거버넌스, 컨소시엄 구조).

**2024-2025 트렌드 레이아웃 (5종)**:
- **Bento Grid**: Hero 카드 1개 + 보조 카드 다수로 정보 밀도 높은 모듈형 배치 (한 장 요약, 기능/혜택 묶음).
- **Sankey**: 흐름의 “비중/분배”를 굵기 차로 표현 (예산/리소스/사용자 전환 비중).
- **Z-Pattern**: 시선 흐름(좌상→우상→좌하→우하)에 맞춘 4포인트 내러티브 (문제→해결→구성→성과).
- **Mind Map**: 중심 개념에서 가지로 확장해 주제 분해/브레인스토밍을 지원 (워크숍 아이데이션, 개념 맵).
- **Stacked Progress**: 하나의 바(또는 스택)에서 단계별 진행/성숙도를 누적 표현 (진행률, 달성도, 단계별 완성도).

### Phase 4: 프롬프트 작성

아래 스타일 가이드 및 4블록 템플릿 구조 적용.

```
+-- Step 4-1. 프롬프트 초안 작성
|   +-- INSTRUCTION 블록: 영어로 작성 (렌더링 안 됨)
|   +-- CONFIGURATION 블록: 영어로 작성 (렌더링 안 됨)
|   +-- CONTENT 블록: 선택한 단일 언어로 작성 (렌더링됨)
|   +-- FORBIDDEN 블록: 영어로 작성 (렌더링 안 됨)
|   +-- 선택된 테마 색상 적용
|
+-- Step 4-2. 도메인 키워드 검증
|   +-- 전문용어 보존 여부 확인
|   +-- 일반화(로봇, 장비 등) 대체 금지
|   +-- 영어 힌트가 INSTRUCTION 블록에만 있는지 확인
|   +-- CONTENT 블록에 언어 병기 없는지 확인
|
+-- Step 4-3. 텍스트 밀도 검증
|   +-- 텍스트 요소 20-30개 범위
|   +-- 핵심 메시지 1-3개 전달
|   +-- 정보 계층 구조 명확성
|
+-- Step 4-4. 복합 레이아웃 검증
|   +-- 레이아웃 조합 수 (최대 3개)
|   +-- 각 레이아웃 영역 구분 명확성
|   +-- 전체 시각적 균형
|
+-- Step 4-5. 렌더링 방지 검증 (필수)
    +-- 프롬프트에 "pt", "px" 단위 포함 여부 확인
    +-- 숫자+단위 조합 (예: 14pt, 32px) 완전 제거
    +-- 크기는 "대형/중형/소형" 또는 "large/medium/small"로만 표현
    +-- 테이블에 "크기" 열 대신 "강조도" 또는 생략
    +-- 중국어/일본어 등 비한글 아시아 문자 포함 금지
```

**프롬프트 구조:**

```
==================================================
[INSTRUCTION BLOCK - DO NOT RENDER]
1. Purpose (영어)
2. Visual Concept (영어 - 시각적 메타포)
3. Style (영어)
4. Colors (영어 - 선택된 테마 색상)
5. Lighting & Mood (영어)
6. Resolution (영어)
==================================================

==================================================
[CONFIGURATION BLOCK - DO NOT RENDER]
- Text Specifications (영어)
- Visual Specifications (영어)
- Layout Combination (영어 - 복합 레이아웃인 경우)
==================================================

==================================================
[CONTENT BLOCK - FOR IMAGE RENDERING]
7. 핵심 메시지 (선택한 단일 언어)
8. 도메인 키워드 (선택한 단일 언어)
9. 시각 구성 (선택한 단일 언어)
10. 텍스트 요소 (선택한 단일 언어) ⚠️ 폰트 크기(pt/px) 명시 금지, 내용과 위치만 기술
==================================================

==================================================
[FORBIDDEN ELEMENTS - DO NOT RENDER]
11. Forbidden Elements (영어)
12. 최종 결과물 목표 (선택한 단일 언어)
==================================================
```

## 핵심 원칙 (Seminar Presentation Style)

### 1. Rich Information with Clarity

```
풍부한 정보와 명확성의 균형:
- 한 이미지에 1-3개의 핵심 개념
- 계층적 정보 구조 (대제목 > 중제목 > 소제목 > 라벨)
- 정보 밀도 높지만 가독성 유지
- 5초 안에 핵심 파악, 30초 안에 세부 이해 가능
```

### 2. Professional Presentation Style

```
전문적 발표 스타일:
- 세미나/워크숍 맥락에 적합한 표현
- 청중의 관심과 이해를 동시에 확보
- 시각적 임팩트와 정보 전달의 조화
- 사내 발표 자료 품질 기준
```

### 3. Visual Impact with Density

```
시각적 임팩트와 정보 밀도:
- 텍스트 요소 20-30개 범위
- 복합 레이아웃으로 다층적 정보 전달
- 의도적 색상 대비로 주목도 확보
- 3D 렌더링 + 아이콘 하이브리드
```

### 4. Single Language Consistency

```
단일 언어 일관성:
- 한글 OR 영어 중 하나만 선택
- 언어 병기 절대 금지 (한글(English) 형식 금지)
- 전문 용어도 선택한 언어로 통일
- INSTRUCTION 블록에만 번역 힌트 포함 가능
```

## 텍스트 규격 (필수)

**세미나 발표 원칙: 풍부한 정보와 가독성의 균형**

> ⚠️ **중요**: 아래 크기 규격은 내부 참고용입니다. **프롬프트에 "pt", "px" 단위를 포함하지 마세요.**

| 요소 | 크기 등급 | 최대 글자수 | 프롬프트 표기 |
|:---|:---:|:---:|:---|
| 대제목 | 대형 (XL) | 15자 | "대형" 또는 "prominent" |
| 중제목 | 중형 (L) | 20자 | "중형" 또는 "medium" |
| 소제목 | 소형 (M) | 15자 | "소형" 또는 "small" |
| 라벨 | 최소형 (S) | 10자 | "최소형" 또는 "minimal" |
| 캡션/주석 | **사용 금지** | - | 프롬프트에 포함 금지 |

**텍스트 밀도 제한:**
- 이미지 전체 텍스트 요소: **20-30개 (최대 30개)**
- 한 영역 내 라벨: **최대 6개**
- 문장 사용 가능 (간결한 경우), 키워드 우선
- **폰트 크기 숫자 (14pt, 32px 등) 프롬프트 포함 절대 금지**

## 복합 레이아웃 가이드

세미나 발표 자료는 한 슬라이드에 여러 개념을 동시에 전달할 수 있습니다.

**조합 규칙:**
- 최대 3개 메타포 조합 허용
- 각 메타포는 명확히 구분된 영역 배치
- 전체 구성의 시각적 균형 유지
- 색상 테마는 전체 일관성 유지

**조합 예시:**
- Hub-Network (중앙) + Card-Grid (하단)
- Section-Flow (상단) + 네트워크 (중앙) + 표 (하단)
- 플로우 (좌측) + 구조 (우측)
- 중심 메타포 (상단) + 3개 그룹 (하단)

**조합 시 주의사항:**
- 주 메타포 1개 + 보조 메타포 1-2개 구조 권장
- 각 메타포 영역 크기 비율: 주 60-70%, 보조 30-40%
- 레이아웃 간 시각적 연결성 확보 (화살표, 선 등)

## 배치 모드

여러 슬라이드를 일괄 생성할 때:

**테마 일관성:**
- 색상 팔레트는 동일하게 유지 (테마 일관성)
- 주조색, 보조색, 강조색, 배경색 고정

**레이아웃 유연성:**
- 레이아웃은 각 개념에 맞게 다를 수 있음
- 슬라이드 1: Hub-Network

## 도메인 키워드 처리

### 보존 원칙

```
금지 패턴 (일반화 금지):
- "굴착기" -> "로봇" (X)
- "휠레그-4족보행로봇" -> "보행 로봇" (X)
- "트랙터" -> "농기계" (X)

허용 패턴 (약어화):
- "휠레그-4족보행로봇" -> "4족로봇" (O)
- "트랙터 기반 자율주행" -> "자율주행 트랙터" (O)
```

### 단일 언어 원칙 (⚠️ 중요)

**한글 선택 시:**
```
CONTENT 블록 (렌더링됨):
라벨: "굴착기" 위치: 좌측
라벨: "4족로봇" 위치: 우측
라벨: "트랙터" 위치: 중앙
```

**영어 선택 시:**
```
CONTENT 블록 (렌더링됨):
Label: "Excavator" Position: Left
Label: "Quadruped Robot" Position: Right
Label: "Tractor" Position: Center
```

⚠️ **병기 절대 금지** → "굴착기 (Excavator)" 형식 사용 불가

## 공통 규칙

- **텍스트**: 단일 언어 (한글 OR 영어, 병기 금지)
- **스타일**: 개념적 3D 또는 플랫 아이콘 (주제에 따라)
- **시점**: 정면 또는 약한 등각(Isometric, 15-30도)
- **배경**: 단색 또는 미니멀 그라데이션
- **해상도**: 8K Quality, 16:9 비율
- **금지 (프롬프트에 포함하지 말 것)**:
  - 복잡한 인포그래픽, 과도한 텍스트, 장식 요소
  - ⚠️ **폰트 크기 숫자** (14pt, 32pt, 48px 등) - 프롬프트 내 절대 포함 금지
  - ⚠️ **"pt", "px" 단위**가 포함된 모든 텍스트
  - 중국어(汉字), 일본어(仮名) 등 비한글 아시아 문자
  - 테이블의 "크기" 열 (내용과 위치만 기술)
  - 언어 병기 (한글(English) 형식)

## 렌더링 방지 체크리스트

프롬프트 완성 후 아래 항목이 포함되어 있지 **않은지** 반드시 확인:

| 금지 패턴 | 예시 | 대체 표현 |
|:---|:---|:---|
| 폰트 크기 숫자+단위 | `14pt`, `32px`, `48pt` | `large`, `medium`, `small` |
| 테이블의 크기 열 | `\| 크기 \|` 열 | 제거 또는 "강조도" 열로 대체 |
| 픽셀 단위 | `8px`, `25px` | `thin`, `thick`, `standard` |
| 퍼센트 단위 | `50%`, `100%` | `half`, `full` |
| 설정 텍스트 | `DO NOT RENDER` | 섹션 제목으로만 사용 |
| 비한글 아시아 문자 | `汎用`, `技术` | 해당 없음 (제거) |
| 언어 병기 | `굴착기 (Excavator)` | 단일 언어만 사용 |

**프롬프트 검증 방법:**
```
프롬프트 내용에서 "\d+pt" 또는 "\d+px" 패턴 검색 → 0건이어야 함
프롬프트 내용에서 "\([A-Z][a-z]+\)" 패턴 검색 → 0건이어야 함 (병기 검증)
```

**올바른 텍스트 요소 테이블 형식:**
```markdown
### 텍스트 요소
| 요소 | 내용 | 위치 |
|------|------|------|
| 메인 타이틀 | Knowledge Curation Platform | 중앙 상단 |
| 서브 타이틀 | AI-Powered Content Discovery | 중앙 |
| 도메인 라벨 | Research / Education / Industry | 각 영역 상단 |
```
> ⚠️ "크기" 열 없음 - 내용과 위치만 기술
