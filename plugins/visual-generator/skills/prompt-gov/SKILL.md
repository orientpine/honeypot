# 슬라이드 프롬프트 생성기 (Gov / Public)

연구 계획서·사업 계획서 문서를 분석해 핵심 메시지를 추출하고, **정부/공공기관 발표 자료**에 적합한 신뢰감 있는 시각자료(슬라이드) 이미지 프롬프트를 작성합니다.

- 스타일: 정부/공공기관·국책 R&D 발표용(공공성, 신뢰감, 과장 금지)
- 텍스트 밀도: **슬라이드당 약 25개(최대 25개) 텍스트 요소 유지**
- 렌더링 안정성: **프롬프트에 숫자+단위(pt/px/% 등) 노출 최소화**, 설정/힌트는 DO NOT RENDER 블록에 격리

## 사용자 입력 스키마

### 필수 입력

| 항목 | 설명 | 예시 |
|------|------|------|
| 입력 문서 | 분석할 연구 계획서/사업 계획서 경로 | `연구계획서.md`, `사업계획서.pdf` |

### 선택 입력

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 무드 테마 | 9가지 테마 중 선택 | `technical-report` |
| 시각화 유형 | 선호 레이아웃(공유 레이아웃 타입) | 자동 선택 |
| 도메인 키워드 | 보존해야 할 고유명사/전문용어 목록 | - |
| 언어 선택 | 한글 또는 영어(단일 언어) | 한글 |
| 출력 폴더 | 프롬프트 저장 위치 | `slide_image_gen/prompts/` |
| 슬라이드 수 | 생성할 슬라이드 개수 | 문서에서 자동 추출 |

## 워크플로우

### Phase 1: 문서 분석

```
+-- Step 1-1. 입력 문서 읽기
|   +-- 연구 계획서, 사업 계획서, 기술 문서 등
|   +-- 문서 구조 파악 (목차, 섹션)
|
+-- Step 1-2. 핵심 개념 추출
|   +-- 비전/목표
|   +-- 세부 기술/기능
|   +-- 핵심 메커니즘
|   +-- 단계별 계획
|   +-- 추진체계/협력 구조
|
+-- Step 1-3. 슬라이드 구성 계획
    +-- 개념별 슬라이드 제목 결정
    +-- 각 슬라이드 레이아웃(시각화 유형) 선택
    +-- 슬라이드 순서/스토리라인 정리
```

### Phase 2: 테마 선택 (9 Themes)

```
+-- Step 2-1. 문서 성격/분야 키워드 분석
|   +-- 정책/공공/국책 → technical-report
|   +-- 성장/성과/확대 → growth
|   +-- 산업/인프라/제조 → clarity
|   +-- 협력/연계/거버넌스 → connection
|   +-- 환경/에너지/지속가능 → innovation
|   +-- ICT/AI/데이터 → tech-focus
|   +-- 지식/아카이빙/플랫폼 → knowledge
|   +-- 발표/보고/브리핑 → presentation
|   +-- 실습/현장/운영 → workshop
|
+-- Step 2-2. 테마 색상 로드
    +-- 아래 스타일 가이드에서 팔레트 확정 (주조/보조/강조/배경)
```

### Phase 3: 시각화 유형(레이아웃) 선택

공유 레이아웃 타입은 `layout-types` 스킬을 참조합니다.

- 목적: "무엇을 보여줄지"에 가장 잘 맞는 구조를 선택
- 원칙: 한 장에서 이해되는 **구조/흐름/관계/비교/로드맵** 중심

### Phase 4: 프롬프트 작성 (4-Block Format)

아래 스타일 가이드를 적용하고 `assets/output_template/prompt_template.md`를 기반으로 작성합니다.

#### [INSTRUCTION BLOCK]
- 영어로 작성 (렌더링 방지)
- 목적/톤앤매너/스타일/컬러/해상도 등 **AI 지시사항**
- **번역 힌트(도메인 키워드 매핑)는 여기에서만 허용**

#### [CONFIGURATION BLOCK]
- 레이아웃: 선택한 레이아웃 타입
- 테마: 선택한 9개 테마 중 1개
- 텍스트 크기: **대형/중형/소형 등급만 사용**
- 텍스트 요소 수: **약 25개(최대 25개)**

#### [CONTENT BLOCK]
- 선택한 **단일 언어**로만 작성
- **언어 병기 금지**: `한글 (English)` / `English (한글)` 형식 절대 금지
- 레이아웃에 맞춘 제목/섹션/라벨/키워드/표(필요 시) 구성

#### [FORBIDDEN ELEMENTS]
- 프롬프트에 포함하면 이미지에 “그대로 렌더링되는” 위험 요소를 명시적으로 금지

### Phase 5: 렌더링 방지 검증 (필수)

```
+-- Step 5-1. 4블록 구조 검증
|   +-- INSTRUCTION/CONFIGURATION/CONTENT/FORBIDDEN 블록이 모두 존재
|   +-- 블록 제목이 그대로 유지됨
|
+-- Step 5-2. 언어 검증
|   +-- CONTENT 블록에 병기(괄호 번역) 0건
|   +-- 번역 힌트는 INSTRUCTION 블록에만 존재
|
+-- Step 5-3. 단위/숫자 패턴 검증
|   +-- CONTENT 블록에 숫자+단위(pt/px/%/alpha 등) 0건
|   +-- 크기 표기는 대형/중형/소형(또는 prominent/medium/small)로만
|
+-- Step 5-4. 텍스트 밀도 검증
    +-- 텍스트 요소 총합 약 25개(최대 25개)
    +-- 과도한 문장형 설명 제거, 키워드 중심 유지
```

## 렌더링 방지 체크리스트

프롬프트 완성 후 아래 항목이 **CONTENT 블록에 존재하지 않는지** 확인합니다.

| 체크 항목 | 금지 예시 | 대체 방법 |
|---|---|---|
| 언어 병기 | `한글 (English)` | 단일 언어로만 작성, 번역 힌트는 INSTRUCTION으로 이동 |
| 크기 단위 노출 | 숫자+단위 조합 | `대형/중형/소형` 등급으로만 표현 |
| 투명도/비율 노출 | `70%`, `alpha` | 별도의 HEX 팔레트(불투명도 변형 색) 사용 또는 생략 |
| 설정 텍스트 렌더링 | “DO NOT RENDER” 문장이 본문에 섞임 | 블록 제목으로만 사용 |
| 과도한 주석/캡션 | 장문의 설명 문장 | 키워드/짧은 구문 중심 |

## 스타일 가이드 (정부/공공기관)

### 프롬프트 구조 (4블록)

모든 `prompt-gov` 프롬프트는 아래 4개 블록으로 구성합니다.

```
[INSTRUCTION BLOCK]
- 목적/톤/스타일/색상/렌더링 힌트(영어 권장)

[CONFIGURATION BLOCK]
- 레이아웃 타입/텍스트 등급/정보 밀도 제한(영어 권장)

[CONTENT BLOCK]
- 이미지에 실제로 표시될 텍스트/라벨/표(단일 언어)

[FORBIDDEN ELEMENTS]
- 절대 금지 요소/패턴(영어 권장)
```

### 언어 규칙 (중요)

- CONTENT 블록: **단일 언어만** 사용(기본: 한글)
  - 금지: 언어 병기(한글(English), English(한글))
  - 금지: 번역 힌트/설정 문장/"DO NOT RENDER" 같은 메타 문구
- INSTRUCTION/CONFIGURATION 블록: 번역 힌트/영어 설명 **허용**
  - 단, "이미지에 번역을 렌더링하지 말 것"을 명시

권장 패턴(렌더링 방지):
- 용어 번역 힌트는 INSTRUCTION에만 작성
- CONTENT에는 최종 라벨(한글 또는 영어)만 둡니다.

### 공통 톤앤매너

- 신뢰감, 공공성, 기술 선도, 미래지향
- National Research Institute & Government Report Style
- 과장 없는 기술 중심 표현(정책/성과는 명확하고 담백하게)
- 비전공자도 이해 가능한 시각적 설명

### 공통 스타일

- 정돈된 산업/기술 시각화 + 미니멀 인포그래픽
- 약한 Isometric 또는 정면 기반(과도한 원근 금지)
- 산세리프 계열, 텍스트 위계 명확
- 배경은 밝게, 대비는 충분히

### 렌더링 방지 체크리스트

프롬프트 최종 저장 전 아래 항목을 반드시 확인합니다.

#### 구조 검증
- [ ] 4블록 구조(Instruction/Configuration/Content/Forbidden)로 분리되어 있음
- [ ] 메타/설정 문장은 CONTENT 블록에 없음

#### 언어 검증
- [ ] CONTENT 블록에 언어 병기 없음
- [ ] 번역/용어 힌트는 INSTRUCTION 블록에만 존재

#### 단위/패턴 검증
- [ ] 숫자+단위(pt/px) 형태의 표기가 없음
- [ ] 테이블에 "크기" 열이 없음(내용/위치 중심)

### Forbidden Elements 작성 규칙

- FORBIDDEN 블록은 영어 권장(렌더링 방지)
- 금지 요소는 "이미지에 포함 금지"로 명확히 선언
- 금지 패턴은 예시가 필요하면 "숫자+단위(pt/px)"처럼 **숫자 없는 형태**로만 설명
