# 개념 시각화 유형

개념의 성격에 따라 적절한 시각적 메타포를 선택합니다.
각 유형은 핵심 원칙, 시각 구성, 적합/부적합 케이스를 포함합니다.

**핵심 원칙**: 모든 유형에서 텍스트는 12-15개 범위로 유지하고, 시각적 메타포가 개념을 전달해야 합니다.

---

## 1. 플로우 메타포 (Flow Metaphor)

**적합한 개념**: 과정, 변환, 단계, 파이프라인, 워크플로우

### 핵심 아이디어

```
"무언가가 A에서 B로 변화/이동한다"

입력 → [변환] → 출력
시작 → 과정 → 결과
원료 → 가공 → 제품
```

### 시각 구성

```
     ┌─────┐        ┌─────┐        ┌─────┐
     │  A  │───────▶│  B  │───────▶│  C  │
     └─────┘        └─────┘        └─────┘
       ↑              ↑              ↑
     시작           변환           결과
```

### 시각화 원칙

- **방향성**: 좌→우 또는 상→하 일관된 흐름
- **연결**: 화살표로 명확한 이동 표현
- **단계 수**: 3-5단계 권장 (너무 많으면 복잡)
- **중간 강조**: 핵심 변환점에 시각적 강조

### 스타일 옵션

**3D 스타일 (물리적 과정):**
```
- 파이프/컨베이어 벨트 메타포
- 물 흐름, 공장 라인
- 실제 물체의 이동/변환
```

**아이콘 스타일 (추상적 과정):**
```
- 깔끔한 도형과 화살표
- 단계별 아이콘
- 미니멀한 연결선
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 3-5개 |
| 라벨 글자수 | 8자 이내 |
| 텍스트 요소 | 10-12개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 데이터 처리 파이프라인
- 머신러닝 학습 과정
- 제품 개발 단계
- 사용자 여정

### 부적합 케이스

- 순환 프로세스 (→ 순환 메타포)
- 계층 구조 (→ 구조 메타포)

## 2. 구조 메타포 (Structure Metaphor)

**적합한 개념**: 계층, 구성, 아키텍처, 레이어, 스택

### 핵심 아이디어

```
"여러 층/부분이 쌓여서 전체를 이룬다"

    [상위]
   --------
    [중간]
   --------
    [기반]
```

### 시각 구성

```
         ┌─────────────────┐
         │    응용 계층     │
         ├─────────────────┤
         │   서비스 계층    │
         ├─────────────────┤
         │   플랫폼 계층    │
         ├─────────────────┤
         │    기반 계층     │
         └─────────────────┘
```

### 시각화 원칙

- **층 구분**: 명확한 경계선 또는 색상 차이
- **위계**: 상단이 상위, 하단이 기반
- **깊이감**: 3D로 표현 시 층간 간격으로 입체감

### 스타일 옵션

**3D 스타일 (물리적 구조):**
```
- 블록 쌓기, 레고 스타일
- 건축물 단면도
- 기기 분해도
```

**아이콘 스타일 (개념적 구조):**
```
- 수평 레이어 다이어그램
- 동심원 (중심이 핵심)
- 네스티드 박스
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 층 수 | 3-5개 |
| 층별 라벨 | 1-2개 |
| 텍스트 요소 | 8-10개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 소프트웨어 아키텍처
- OSI 7 레이어
- 기술 스택
- 조직 구조

### 부적합 케이스

- 프로세스 흐름 (→ 플로우 메타포)
- 네트워크 관계 (→ 네트워크 메타포)

## 3. 네트워크 메타포 (Network Metaphor)

**적합한 개념**: 관계, 연결, 네트워크, 생태계, 상호작용

### 핵심 아이디어

```
"여러 요소가 서로 연결되어 있다"

    ○───────○
    │       │
    │   ○   │
    │  /│\  │
    ○─○ ○ ○─○
```

### 시각 구성

```
        ┌───┐
        │ A │──────────┐
        └─┬─┘          │
          │      ┌─────┴─────┐
        ┌─┴─┐    │     C     │
        │ B │────┤           │
        └───┘    └─────┬─────┘
                       │
                    ┌──┴──┐
                    │  D  │
                    └─────┘
```

### 시각화 원칙

- **노드**: 각 요소를 명확한 형태로
- **연결선**: 관계의 강도/유형을 선 스타일로
- **중심성**: 핵심 노드는 크기/색상으로 강조
- **배치**: 관련 노드는 가깝게

### 스타일 옵션

**3D 스타일 (물리적 네트워크):**
```
- 구체와 막대 연결
- 분자 구조 스타일
- 3D 공간에 배치된 노드
```

**아이콘 스타일 (개념적 네트워크):**
```
- 원형 노드와 선
- 방사형 배치
- 클러스터 그룹화
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 노드 수 | 4-8개 |
| 노드 라벨 | 6자 이내 |
| 텍스트 요소 | 12-15개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- API 연동 구조
- 마이크로서비스 아키텍처
- 이해관계자 관계
- 기술 생태계

### 부적합 케이스

- 순차적 프로세스 (→ 플로우 메타포)
- 계층 구조 (→ 구조 메타포)

## 4. 대비 메타포 (Contrast Metaphor)

**적합한 개념**: 비교, 대조, 전후, 차이, 선택

### 핵심 아이디어

```
"A와 B의 차이/변화를 보여준다"

    [Before]  vs  [After]
    [Old]     →   [New]
    [A방식]   ↔   [B방식]
```

### 시각 구성

```
  ┌─────────────┐     ┌─────────────┐
  │             │     │             │
  │   BEFORE    │  →  │    AFTER    │
  │             │     │             │
  │  (문제점)   │     │   (해결)    │
  │             │     │             │
  └─────────────┘     └─────────────┘
       연한색              진한색
```

### 시각화 원칙

- **대칭**: 동일한 구도, 다른 내용
- **색상 대비**: 부정적(연함) vs 긍정적(진함)
- **중앙 변환**: 화살표 또는 전환 표시
- **명확한 차이**: 한눈에 변화 인식

### 스타일 옵션

**3D 스타일 (물리적 대비):**
```
- 같은 물체의 두 상태
- 분할 화면 (좌/우)
- 시간에 따른 변화
```

**아이콘 스타일 (개념적 대비):**
```
- 좌우 또는 상하 분할
- X표시 vs 체크표시
- 색상으로 구분
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 비교 항목 | 2개 (A vs B) |
| 세부 요소 | 각 3-4개 |
| 텍스트 요소 | 10-12개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 기존 vs 신규 방식
- 문제 vs 해결책
- 경쟁 제품 비교
- 적용 전후 변화

### 부적합 케이스

- 3개 이상 비교 (→ 그룹 메타포)
- 순차적 변화 (→ 플로우 메타포)

## 5. 진화 메타포 (Evolution Metaphor)

**적합한 개념**: 성장, 발전, 단계적 향상, 학습, 성숙

### 핵심 아이디어

```
"작은 것에서 큰 것으로, 단순에서 복잡으로"

    ● → ●● → ●●● → ★
   씨앗  새싹  성장  결실
```

### 시각 구성

```
                                    ★★★
                            ★★     ●●●
                    ★      ●●      ●●
            ●      ●●      ●       ●
    ─────●──●●─────●●●────●●●●────★★★★★─────▶
        1단계    2단계    3단계    4단계
```

### 시각화 원칙

- **크기 변화**: 작은 것에서 큰 것으로
- **복잡도 증가**: 단순에서 정교함으로
- **방향성**: 좌→우 또는 하→상 성장
- **자연스러움**: 급격한 변화보다 점진적

### 스타일 옵션

**3D 스타일 (물리적 성장):**
```
- 식물 성장 메타포
- 건물 건설 단계
- 결정/결실 형성
```

**아이콘 스타일 (개념적 성장):**
```
- 크기가 커지는 도형
- 레벨업 표시
- 진행 바 / 계단
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 3-5개 |
| 단계 라벨 | 6자 이내 |
| 텍스트 요소 | 8-10개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 기술 성숙도 (TRL)
- 학습 곡선
- 제품 발전 로드맵
- 역량 성장 단계

### 부적합 케이스

- 순환 프로세스 (→ 순환 메타포)
- 병렬 비교 (→ 대비 메타포)

## 6. 중심 메타포 (Core Metaphor)

**적합한 개념**: 핵심, 중심, 코어, 플랫폼, 1:N 관계

### 핵심 아이디어

```
"하나의 중심에서 여러 것이 뻗어나간다"

        ○
        │
    ○───●───○
        │
        ○
```

### 시각 구성

```
              ┌─────┐
              │ App │
              └──┬──┘
                 │
    ┌─────┐   ┌──┴──┐   ┌─────┐
    │ API │───│CORE │───│ DB  │
    └─────┘   └──┬──┘   └─────┘
                 │
              ┌──┴──┐
              │ UI  │
              └─────┘
```

### 시각화 원칙

- **중심 강조**: 가장 크고 눈에 띄게
- **방사 배치**: 중심에서 균등하게 뻗어나감
- **연결 명확**: 중심과 주변의 관계 표시
- **주변 균형**: 외곽 요소들의 크기/간격 균등

### 스타일 옵션

**3D 스타일 (물리적 중심):**
```
- 태양계 모델
- 허브와 스포크
- 중앙 플랫폼
```

**아이콘 스타일 (개념적 중심):**
```
- 방사형 다이어그램
- 마인드맵 스타일
- 중심 원형 배치
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 주변 요소 수 | 4-6개 |
| 중심 라벨 | 8자 이내 |
| 주변 라벨 | 6자 이내 |
| 텍스트 요소 | 10-12개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 플랫폼 아키텍처
- 핵심 기술과 응용
- 중앙 집중 시스템
- 허브 앤 스포크 모델

### 부적합 케이스

- 순차적 흐름 (→ 플로우 메타포)
- 계층 구조 (→ 구조 메타포)

## 7. 순환 메타포 (Cycle Metaphor)

**적합한 개념**: 반복, 피드백, 사이클, 지속적 개선, 루프

### 핵심 아이디어

```
"끝없이 반복되는 과정"

    ┌→ A → B → C ─┐
    │             │
    └── D ← ─────┘
```

### 시각 구성

```
           ┌─────┐
      ┌────│  A  │────┐
      │    └─────┘    │
      ▼               ▼
   ┌─────┐         ┌─────┐
   │  D  │         │  B  │
   └─────┘         └─────┘
      ▲               │
      │    ┌─────┐    │
      └────│  C  │◀───┘
           └─────┘
```

### 시각화 원칙

- **순환 표시**: 닫힌 화살표 루프
- **방향 일관**: 시계방향 또는 반시계방향
- **단계 균등**: 각 단계 크기 동일
- **시작점 표시**: 선택적으로 강조

### 스타일 옵션

**3D 스타일 (물리적 순환):**
```
- 회전하는 기어
- 물 순환 사이클
- 재활용 심볼
```

**아이콘 스타일 (개념적 순환):**
```
- 원형 화살표
- 단계별 아이콘 배치
- 순환 다이어그램
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 3-6개 |
| 단계 라벨 | 8자 이내 |
| 텍스트 요소 | 10-12개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- DevOps 사이클
- 애자일 스프린트
- PDCA 사이클
- 데이터 파이프라인

### 부적합 케이스

- 일방향 프로세스 (→ 플로우 메타포)
- 계층 구조 (→ 구조 메타포)

## 8. 그룹 메타포 (Group Metaphor)

**적합한 개념**: 분류, 카테고리, 종류, 옵션, 구성원

### 핵심 아이디어

```
"여러 종류/유형이 있다"

    [타입 A]  [타입 B]  [타입 C]
```

### 시각 구성

```
    ┌─────────────────────────────────────┐
    │            [상위 개념]              │
    │  ┌───────┐ ┌───────┐ ┌───────┐     │
    │  │ 유형A │ │ 유형B │ │ 유형C │     │
    │  │       │ │       │ │       │     │
    │  │ ●●●   │ │ ■■■   │ │ ▲▲▲   │     │
    │  └───────┘ └───────┘ └───────┘     │
    └─────────────────────────────────────┘
```

### 시각화 원칙

- **명확한 구분**: 각 그룹 영역 분리
- **일관된 형식**: 같은 레벨은 같은 크기
- **대표 아이콘**: 각 카테고리 특성 표현
- **균형 배치**: 그룹 간 균등한 공간

### 스타일 옵션

**3D 스타일 (물리적 그룹):**
```
- 선반 위 물건들
- 컬렉션 진열
- 박스 분류
```

**아이콘 스타일 (개념적 그룹):**
```
- 카드 레이아웃
- 그리드 배치
- 탭/섹션 구분
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 그룹 수 | 2-4개 |
| 그룹 라벨 | 8자 이내 |
| 그룹별 예시 | 1-3개 |
| 텍스트 요소 | 12-15개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 제품 라인업
- 서비스 종류
- 알고리즘 분류
- 사용자 유형

### 부적합 케이스

## 9. 동심원 메타포 (Core-Shell Metaphor) [신규]

**적합한 개념**: 계층적 포함 관계, 핵심-확장 구조, 보안 레이어, 추상화 수준

```
"안에서 밖으로 확장되는 레이어 구조"

       ┌───────────────────┐
       │   ┌───────────┐   │
       │   │   ┌───┐   │   │
       │   │   │ ● │   │   │
       │   │   └───┘   │   │
```

```
            ┌─────────────────────────────────┐
            │         응용 레이어             │

            │   │   │   │  Core   │   │   │   │
            │   │   │   │   AI    │   │   │   │
            │   │   │   └─────────┘   │   │   │
            │   │   └─────────────────┘   │   │
            │   └─────────────────────────┘   │
            └─────────────────────────────────┘
                          ↑
               메인(중심) → 서브 → 라이트(외곽)
```

### 시각화 원칙

- **중심에서 외곽으로 색상 연해짐**
- **각 레이어 경계 명확**
- **레이어별 라벨 배치**
- **화살표로 의존성 표현 가능**

### 스타일 옵션


**아이콘 스타일:**
```
- 동심원 다이어그램
- 중첩 박스
- 레이어 스택
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 레이어 수 | 3-5개 |
| 레이어 라벨 글자수 | 10자 이내 |
| 부가 설명 | 레이어당 1줄 |
| 텍스트 요소 | 8-12개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 소프트웨어 아키텍처 레이어

## 10. Swimlane 메타포 (다중주체 프로세스) [신규]

**적합한 개념**: 다중 주체 병렬 프로세스, 역할별 워크플로우, 협업 과정

### 핵심 아이디어

```
"여러 주체가 동시에 다른 역할을 수행"

  주체A: ●────────●────────●
  주체B:      ●────────●
  주체C:           ●────────●
```

### 시각 구성

```
         │ 1단계     │ 2단계      │ 3단계     │
─────────┼───────────┼────────────┼───────────┤
         │           │            │           │
 연구원  │  ●──────►─┼───●        │    ●      │
         │  기초연구 │   │        │    ↑      │
─────────┼───────────┼───┼────────┼────┼──────┤
         │           │   ↓        │    │      │
 기업    │           │   ●──────►─┼────●      │
         │           │   시제품   │  상용화   │
─────────┼───────────┼────────────┼───────────┤
         │           │            │           │
 정부    │  ●────────┼────────────┼───●       │
         │  기획     │            │  평가     │
─────────┴───────────┴────────────┴───────────┘
```

### 시각화 원칙

- **레인 간 구분선 명확**
- **레인별 배경색 구분 (연한 톤)**
- **레인 간 이동은 수직 화살표**
- **레인 내 이동은 수평 화살표**

### 스타일 옵션

**3D 스타일:**
```
- 다층 컨베이어 벨트
- 병렬 생산 라인
- 다중 트랙
```

**아이콘 스타일:**
```
- Swimlane 다이어그램
- 간트 차트 스타일
- 병렬 타임라인
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 레인 수 | 2-4개 |
| 단계 수 | 3-5개 |
| 활동 노드 수 | 총 8-15개 |
| 노드 라벨 글자수 | 8자 이내 |
| 텍스트 요소 | 12-15개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 연구-기업-정부 협력 프로세스
- 부서 간 워크플로우
- 다중 이해관계자 프로젝트
- 역할별 업무 분담

### 부적합 케이스

- 단일 주체 프로세스 (→ 플로우 메타포)
- 계층 구조 (→ 구조 메타포)

## 11. 전략맵 메타포 (Strategic Map) [신규]

**적합한 개념**: 2x2/3x3 전략 맵, 포지셔닝, 우선순위 결정, 리스크 매트릭스

### 핵심 아이디어

```
"두 축을 기준으로 항목을 배치"

     높음 ▲
          │  [Q2]    [Q1]
     축Y  │
          │  [Q3]    [Q4]
     낮음 └──────────────▶
            낮음      높음
                축X
```

### 시각 구성 (2x2)

```
     높음 ▲
          │  ┌─────────────┬─────────────┐
          │  │             │             │
     영   │  │   우선순위  │   즉시     │
     향   │  │     낮음    │   실행     │
     도   │  │             │             │
          │  ├─────────────┼─────────────┤
          │  │             │             │
          │  │    보류     │   계획     │
          │  │   (제외)    │   수립     │
          │  │             │             │
     낮음 │  └─────────────┴─────────────┘
          └────────────────────────────────►
                낮음              높음
                      긴급도
```

### 시각화 원칙

- **축 라벨 명확하게 표시**
- **사분면별 색상 구분**
- **핵심 영역 강조**
- **항목 배치로 포지셔닝 표현**

### 스타일 옵션

**3D 스타일:**
```
- 평면 위 배치된 객체들
- 높이로 중요도 표현
- 지형도 스타일
```

**아이콘 스타일:**
```
- 2x2/3x3 그리드
- 버블 차트 스타일
- 매트릭스 다이어그램
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 그리드 크기 | 2x2 또는 3x3 |
| 축 라벨 글자수 | 8자 이내 |
| 영역 제목 글자수 | 10자 이내 |
| 배치 항목 수 | 4-12개 |
| 텍스트 요소 | 10-15개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 우선순위 매트릭스
- 리스크 평가
- 기술 포트폴리오 분석
- 시장 포지셔닝

### 부적합 케이스

- 프로세스 흐름 (→ 플로우 메타포)

## 12. 깔때기 메타포 (Funnel Metaphor) [신규]

**적합한 개념**: 단계별 축소, 선별 과정, 전환율, 파이프라인

### 핵심 아이디어

```
"위에서 아래로 점점 좁아지며 선별"

     ╔═════════════════════════╗
     ║         입력 (100%)     ║
     ╚═══════════╤═════════════╝
           ╔═════╧═════╗
           ║  1차 (50%) ║
           ╚═════╤═════╝
              ╔══╧══╗
              ║최종 ║
              ╚════╝
```

### 시각 구성

```
        ┌───────────────────────────────────┐
        │         1,000건 접수              │
        │              (100%)               │
        └─────────────────┬─────────────────┘
              ┌───────────┴───────────┐
              │     500건 1차 심사     │
              │         (50%)         │
              └───────────┬───────────┘
                    ┌─────┴─────┐
                    │ 100건 선정 │
                    │   (10%)   │
                    └─────┬─────┘
                       ┌──┴──┐
                       │ 50건 │
                       │최종  │
                       └─────┘
                        (5%)
```

### 시각화 원칙

- **위에서 아래로 자연스럽게 좁아짐**
- **각 단계 수치/비율 표시**
- **단계별 색상 변화 (연함→진함)**
- **화살표로 흐름 표현**

### 스타일 옵션

**3D 스타일:**
```
- 실제 깔때기 형태
- 계단식 축소
- 파이프라인 구조
```

**아이콘 스타일:**
```
- 역삼각형 구조
- 단계별 박스 축소
- 플로우 다이어그램
```

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 3-5개 |
| 단계 라벨 글자수 | 10자 이내 |
| 수치 표시 | 필수 |
| 텍스트 요소 | 8-12개 |
| 도메인 키워드 | 축약 허용, 대체 금지 |

### 적합 케이스

- 사업화 파이프라인
- 선정 프로세스
- 고객 전환 퍼널
- 과제 선발 과정

### 부적합 케이스

- 병렬 프로세스 (→ Swimlane 메타포)
- 순환 구조 (→ 순환 메타포)
