# 슬라이드 레이아웃 유형

개념 유형에 따라 적절한 시각화 레이아웃 선택. 각 레이아웃은 ASCII 예시, 권장 사양, 색상 배치 전략을 포함.

---

## 1. 계층형 피라미드

**적합한 개념**: 비전/목표, 단계별 발전, 우선순위 구조, 가치 체계

### 구조
- 최상단: 최종 비전 (가장 좁음, 메인 컬러)
- 중단: 상위 목표 (중간 너비, 서브 컬러)
- 하단: 기반 기술/단계 (가장 넓음, 배경 컬러)

### ASCII 예시
```
                    ┌─────────┐
                    │  비전   │  ← 메인 컬러
                    └────┬────┘
               ┌─────────┴─────────┐
               │    상위 목표 1-3   │  ← 서브 컬러
               └────────┬──────────┘
        ┌───────────────┴───────────────┐
        │     기반 기술 / 핵심 역량      │  ← 라이트 컬러
        └───────────────────────────────┘
```

### 시각화 원칙
- 색상 그라데이션 (하단→상단: 연한→진한)
- 층 간 구분선 또는 미묘한 그림자
- 위쪽 방향 화살표 (선택적)
- 각 층 내부에 아이콘 배치 가능

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 최대 층수 | 3-5층 |
| 층당 라벨 | 1-3개 |
| 라벨 최대 글자수 | 15자 |
| 텍스트 밀도 | 낮음 |
| 최소 폰트 크기 | 16pt |

### 색상 배치 전략
- **메인**: 최상단 비전 박스
- **서브**: 중간 목표 층
- **강조**: 핵심 키워드 또는 화살표
- **배경**: 하단 기반 층

### 적합 케이스
- 연구 비전 및 최종 목표 표현
- 기술 성숙도 단계 (TRL)
- 조직 가치 체계

### 부적합 케이스
- 순환 프로세스 (시작/끝 없음)
- 병렬 비교 (좌우 대칭 필요)

### 주의사항
- 3층 초과 시 가독성 저하
- 각 층의 텍스트는 키워드 수준으로 제한
- 층 간 높이 비율 일정하게 유지

## 2. 중앙 방사형 (Hub-Spoke)

**적합한 개념**: 통합 플랫폼, 1:N 관계, 중앙 집중 시스템, 코어 기술

### 구조
- 중앙: 핵심 노드 (가장 크고 진한 색상)
- 외곽: 연결된 요소들 (균등 배치)
- 연결선: 중앙에서 외곽으로 방사

### ASCII 예시
```
              ┌─────────┐
              │ 모듈 A  │
              └────┬────┘
                   │
     ┌─────────┐   │   ┌─────────┐
     │ 모듈 E  │───┼───│ 모듈 B  │
     └─────────┘   │   └─────────┘
                   │
            ┌──────┴──────┐
            │             │
            │  핵심 코어  │  ← 메인 컬러 (중앙)
            │             │
            └──────┬──────┘
                   │
     ┌─────────┐   │   ┌─────────┐
     │ 모듈 D  │───┴───│ 모듈 C  │
     └─────────┘       └─────────┘
```

### 시각화 원칙
- 중앙 노드 강조 (크기 1.5배, 글로우 효과)
- 외곽 노드 균등 배치 (60도, 72도, 90도 간격)
- 데이터 흐름 표현 (점선 또는 애니메이션 화살표)
- 양방향/단방향 화살표 선택

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 외곽 노드 수 | 4-8개 |
| 중앙 라벨 글자수 | 10자 이내 |
| 외곽 라벨 글자수 | 12자 이내 |
| 텍스트 밀도 | 중간 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 중앙 핵심 노드
- **서브**: 외곽 노드들 (동일 색상)
- **강조**: 특정 연결선 또는 핵심 외곽 노드
- **배경**: 전체 배경

### 적합 케이스
- 통합 플랫폼 아키텍처
- AI 모델과 응용 분야 관계
- 핵심 기술과 파생 기술

### 부적합 케이스
- 순차적 프로세스 (순서가 중요한 경우)
- 계층 구조 (상하 관계)

### 주의사항
- 외곽 노드 8개 초과 시 혼잡
- 연결선이 교차하지 않도록 배치
- 중앙 노드 텍스트는 짧게

## 3. 원형 순환 (Flywheel)

**적합한 개념**: 파이프라인, 반복 프로세스, 피드백 루프, 지속 개선

### 구조
- 중앙: 허브 (플랫폼/코어) - 선택적
- 외곽: 순환 단계 노드
- 화살표: 시계 방향 순환

### ASCII 예시
```
                  ┌─────────┐
            ┌────→│ 1. 수집 │────┐
            │     └─────────┘    │
            │                    ↓
      ┌─────┴───┐          ┌─────┴───┐
      │4. 배포  │          │2. 분석  │
      └─────┬───┘          └─────┬───┘
            │                    │
            │     ┌─────────┐    │
            └────←│ 3. 학습 │←───┘
                  └─────────┘
                       
           [중앙: AI 플랫폼]  ← 선택적
```

### 시각화 원칙
- 각 단계별 고유 색상 또는 번호
- 회전 효과 (곡선 화살표)
- 단계 간 연결 명확화
- 외부 입력/출력 별도 표시

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 순환 단계 수 | 3-6개 |
| 단계 라벨 글자수 | 10자 이내 |
| 부가 설명 | 단계당 1줄 |
| 텍스트 밀도 | 중간 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 핵심 단계 또는 중앙 허브
- **서브**: 일반 단계 노드들
- **강조**: 시작점 또는 핵심 단계
- **배경**: 전체 배경

### 적합 케이스
- 데이터 파이프라인 (수집→분석→학습→배포)
- MLOps 사이클
- 지속적 개선 프로세스

### 부적합 케이스
- 일회성 프로세스
- 계층적 구조

### 주의사항
- 6단계 초과 시 복잡해짐
- 화살표 방향 일관성 유지
- 각 단계 크기 동일하게

## 4. 3D 분해도 (Exploded View)

**적합한 개념**: 모듈 구조, 하드웨어 구성, 시스템 아키텍처, 조립 과정

### 구조
- 중앙: 본체 (완성품 또는 코어)
- 분리: 구성 요소들 (본체에서 떨어진 위치)
- 점선: 장착 위치 연결

### ASCII 예시
```
       ┌─────────────┐
       │   센서 모듈  │ ·····┐
       └─────────────┘      ·
                            ·
   ┌─────────┐         ┌────·────┐         ┌─────────┐
   │ 통신    │ ········│        │·········│  제어   │
   │ 모듈    │         │  본체  │         │  모듈   │
   └─────────┘         │        │         └─────────┘
                       └────┬────┘
                            ·
       ┌─────────────┐      ·
       │  구동 모듈   │ ·····┘
       └─────────────┘
```

### 시각화 원칙
- 약한 Isometric 시점 (15-30도)
- 모듈별 고유 색상
- 깊이감 유지하며 분리
- 점선으로 장착 위치 표시

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 모듈 수 | 4-8개 |
| 모듈 라벨 글자수 | 10자 이내 |
| 부가 설명 | 모듈당 1줄 |
| 텍스트 밀도 | 낮음 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 중앙 본체/코어
- **서브**: 주요 모듈들
- **강조**: 핵심 모듈 또는 신규 개발 부분
- **배경**: 전체 배경 (매우 연한 색)

### 적합 케이스
- 로봇 하드웨어 구성
- 시스템 아키텍처
- 플랫폼 모듈 구조

### 부적합 케이스
- 추상적 개념
- 시간 순서 프로세스

### 주의사항
- 모듈 간 겹침 방지
- 점선이 교차하지 않도록
- Isometric 각도 일관성 유지

## 5. 좌우 병렬 비교

**적합한 개념**: 전후 비교, 실제 vs 시뮬레이션, As-Is/To-Be, 대비 표현

### 구조
- 좌측: A 상태/환경 (현재/실제)
- 우측: B 상태/환경 (목표/가상)
- 중앙: 변환/연결 요소

### ASCII 예시
```
┌─────────────────┐         ┌─────────────────┐
│                 │         │                 │
│    현재 상태    │   ───►  │    목표 상태    │
│    (As-Is)      │   변환  │    (To-Be)      │
│                 │         │                 │
├─────────────────┤         ├─────────────────┤
│ • 수동 작업     │         │ • 자동화        │
│ • 개별 시스템   │         │ • 통합 플랫폼   │
│ • 낮은 효율     │         │ • 고효율        │
└─────────────────┘         └─────────────────┘
     [라이트 톤]                [메인 컬러]
```

### 시각화 원칙
- 동일한 구도/앵글로 양쪽 표현
- 색상으로 명확히 구분
- 중앙에 변환 화살표
- 대응 요소 시각적 연결 (점선)

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 비교 항목 수 | 3-5개 |
| 항목 라벨 글자수 | 15자 이내 |
| 영역 제목 글자수 | 10자 이내 |
| 텍스트 밀도 | 중간 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 목표 상태 (우측) 영역
- **서브**: 현재 상태 (좌측) 영역 - 연한 톤
- **강조**: 변환 화살표, 핵심 변화점
- **배경**: 전체 배경

### 적합 케이스
- 디지털 전환 전후 비교
- 실제 환경 vs 시뮬레이션
- 문제점 vs 해결책

### 부적합 케이스
- 3개 이상 비교
- 순환 프로세스

### 주의사항
- 좌우 영역 크기 동일하게
- 비교 항목 수 동일하게 맞춤
- 중앙 화살표 크기 적절히

## 6. 수평 타임라인

**적합한 개념**: 로드맵, 연도별 계획, 마일스톤, 단계별 진행

### 구조
- 상단: 연도/단계 축 (좌→우: 시간 흐름)
- 하단: 트랙별 마일스톤 노드
- 구분: 단계별 색상 배경

### ASCII 예시
```
     1단계 (2027-2029)          2단계 (2030-2031)
  ┌────────────────────────┐ ┌────────────────────┐
  │   2027    2028    2029 │ │   2030       2031  │
  │    ●───────●───────●   │ │    ●─────────●     │
  │    │       │       │   │ │    │         │     │
  │ ┌──┴──┐ ┌──┴──┐ ┌──┴──┐│ │ ┌──┴──┐   ┌──┴──┐  │
  │ │기반 │ │핵심 │ │통합 ││ │ │실증 │   │상용 │  │
  │ │구축 │ │개발 │ │테스트││ │ │적용 │   │화   │  │
  │ └─────┘ └─────┘ └─────┘│ │ └─────┘   └─────┘  │
  └────────────────────────┘ └────────────────────┘
       [서브 컬러 배경]          [메인 컬러 배경]
```

### 시각화 원칙
- 연도 마커 (원형 노드)
- 트랙별 고유 색상
- 핵심 마일스톤 ★ 표시
- 단계 경계 명확화

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 연도 범위 | 3-7년 |
| 마일스톤 수 | 5-10개 |
| 마일스톤 라벨 글자수 | 10자 이내 |
| 트랙 수 | 1-3개 |
| 텍스트 밀도 | 중간 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 후반기/최종 단계 배경
- **서브**: 초반기/기반 단계 배경
- **강조**: 핵심 마일스톤 노드
- **배경**: 전체 배경

### 적합 케이스
- 연구개발 로드맵
- 프로젝트 일정
- 기술 발전 단계

### 부적합 케이스
- 순환 프로세스
- 계층 구조

### 주의사항
- 연도 간격 균등하게
- 마일스톤 너무 많으면 복잡
- 트랙 3개 초과 시 가독성 저하

## 7. 조직도 + 네트워크 혼합

**적합한 개념**: 추진체계, 협력 관계, 역할 분담, 거버넌스

### 구조
- 상단: 총괄 기관 (가장 큰 노드)
- 중단: 세부 과제/부서 (분기 구조)
- 하단: 참여 기관/역할 (네트워크 연결)

### ASCII 예시
```
                ┌───────────────┐
                │   총괄기관    │  ← 메인 컬러
                │    (KIMM)     │
                └───────┬───────┘
           ┌────────────┼────────────┐
           │            │            │
     ┌─────┴─────┐┌─────┴─────┐┌─────┴─────┐
     │ 세부과제1 ││ 세부과제2 ││ 세부과제3 │ ← 서브 컬러
     └─────┬─────┘└─────┬─────┘└─────┬─────┘
           │            │            │
     ┌─────┴─────┐      │      ┌─────┴─────┐
     │  대학 A   │······│······│  기업 B   │ ← 협력 점선
     └───────────┘      │      └───────────┘
                  ┌─────┴─────┐
                  │  연구원 C  │
                  └───────────┘
```

### 시각화 원칙
- 계층 관계: 굵은 실선
- 협력 관계: 얇은 점선
- 기관 유형별 색상 구분
- 예산/역할 라벨 추가

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 계층 수 | 2-4층 |
| 기관 수 | 5-12개 |
| 기관명 글자수 | 12자 이내 |
| 역할 설명 글자수 | 15자 이내 |
| 텍스트 밀도 | 중간 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 총괄/주관 기관
- **서브**: 세부과제/참여기관
- **강조**: 핵심 역할 또는 예산 표시
- **배경**: 전체 배경

### 적합 케이스
- 연구과제 추진체계
- 컨소시엄 구조
- 협력 네트워크

### 부적합 케이스
- 단순 계층 구조 (조직도만 필요한 경우)
- 시간 순서 프로세스

### 주의사항
- 연결선 교차 최소화
- 기관명 축약어 사용 권장
- 역할 설명은 간결하게

## 8. 표(Matrix) 구조

**적합한 개념**: 분류 체계, 비교 분석, 다차원 정보, 매핑

### 구조
- 열(Column): 분야/카테고리
- 행(Row): 기능/레벨
- 셀: 각 조합의 내용

### ASCII 예시
```
┌─────────┬───────────┬───────────┬───────────┐
│         │   건설    │   농업    │   재난    │
├─────────┼───────────┼───────────┼───────────┤
│ 인지    │ 구조물    │ 작물      │ 피해자    │
│         │ 스캔      │ 모니터링  │ 탐지      │
├─────────┼───────────┼───────────┼───────────┤
│ 판단    │ 안전      │ 생육      │ 위험      │
│         │ 진단      │ 분석      │ 평가      │
├─────────┼───────────┼───────────┼───────────┤
│ 행동    │ 자동      │ 정밀      │ 구조      │
│         │ 시공      │ 방제      │ 작업      │
└─────────┴───────────┴───────────┴───────────┘
    ↑ 행 헤더          ↑ 열 헤더
  (메인 컬러)        (서브 컬러)
```

### 시각화 원칙
- 헤더 행/열 색상 구분
- 셀 내용은 키워드 수준
- 아이콘 활용 가능
- 교차점 강조 가능

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 열 수 | 3-5개 |
| 행 수 | 3-5개 |
| 셀 내용 글자수 | 10자 이내 |
| 헤더 글자수 | 8자 이내 |
| 텍스트 밀도 | 높음 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 행 헤더 배경
- **서브**: 열 헤더 배경
- **강조**: 핵심 셀 또는 대각선
- **배경**: 일반 셀 배경 (매우 연한 색)

### 적합 케이스
- 기술 분류 체계
- 역할-책임 매트릭스
- 기능-분야 매핑

### 부적합 케이스
- 프로세스 흐름
- 계층 구조

### 주의사항
- 5x5 초과 시 가독성 저하
- 셀 내용 길면 줄바꿈 필요
- 헤더 폰트는 본문보다 크게

## 9. 동심원 레이어 (Core-Shell) [신규]

**적합한 개념**: 계층적 포함 관계, 핵심-확장 구조, 보안 레이어, 추상화 수준

### 구조
- 중심: 핵심 (Core)
- 중간: 확장 레이어들
- 외곽: 최외곽 인터페이스/응용

### ASCII 예시
```
            ┌─────────────────────────────────┐
            │         응용 레이어             │
            │   ┌─────────────────────────┐   │
            │   │      서비스 레이어      │   │
            │   │   ┌─────────────────┐   │   │
            │   │   │  플랫폼 레이어  │   │   │
            │   │   │   ┌─────────┐   │   │   │
            │   │   │   │  Core   │   │   │   │
            │   │   │   │   AI    │   │   │   │
            │   │   │   └─────────┘   │   │   │
            │   │   └─────────────────┘   │   │
            │   └─────────────────────────┘   │
            └─────────────────────────────────┘
                          ↑
               메인(중심) → 서브 → 라이트(외곽)
```

### 시각화 원칙
- 중심에서 외곽으로 색상 연해짐
- 각 레이어 경계 명확
- 레이어별 라벨 배치
- 화살표로 의존성 표현 가능

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 레이어 수 | 3-5개 |
| 레이어 라벨 글자수 | 12자 이내 |
| 부가 설명 | 레이어당 1줄 |
| 텍스트 밀도 | 낮음 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 중심 Core
- **서브**: 중간 레이어들 (그라데이션)
- **강조**: 특정 레이어 하이라이트
- **배경**: 최외곽 또는 전체 배경

### 적합 케이스
- 소프트웨어 아키텍처 레이어
- 보안 계층 구조
- 추상화 수준 표현

### 부적합 케이스
- 순차 프로세스
- 병렬 비교

### 주의사항
- 레이어 5개 초과 시 복잡
- 중심 텍스트는 반드시 보이도록
- 레이어 간 간격 균등하게

## 10. Swimlane 프로세스 [신규]

**적합한 개념**: 다중 주체 병렬 프로세스, 역할별 워크플로우, 협업 과정

### 구조
- 수평 레인: 각 주체/역할별 영역
- 수직 축: 시간/단계 흐름
- 노드: 각 레인 내 활동

### ASCII 예시
```
         │ 1단계     │ 2단계      │ 3단계     │
─────────┼───────────┼────────────┼───────────┤
         │           │            │           │
 연구원  │  ●──────►─┼───●        │    ●      │
         │  기초연구 │   │        │    ↑      │
─────────┼───────────┼───┼────────┼────┼──────┤
         │           │   ↓        │    │      │
 기업    │           │   ●──────►─┼────●      │
         │           │   시제품   │  상용화   │
─────────┼───────────┼────────────┼───────────┤
         │           │            │           │
 정부    │  ●────────┼────────────┼───●       │
         │  기획     │            │  평가     │
─────────┴───────────┴────────────┴───────────┘
```

### 시각화 원칙
- 레인 간 구분선 명확
- 레인별 배경색 구분 (연한 톤)
- 레인 간 이동은 수직 화살표
- 레인 내 이동은 수평 화살표

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 레인 수 | 2-4개 |
| 단계 수 | 3-5개 |
| 활동 노드 수 | 총 8-15개 |
| 노드 라벨 글자수 | 8자 이내 |
| 텍스트 밀도 | 중간 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 핵심 주체 레인 배경
- **서브**: 일반 주체 레인 배경
- **강조**: 주요 전환점/핸드오프
- **배경**: 전체 배경

### 적합 케이스
- 연구-기업-정부 협력 프로세스
- 부서 간 워크플로우
- 다중 이해관계자 프로젝트

### 부적합 케이스
- 단일 주체 프로세스
- 계층 구조

### 주의사항
- 레인 4개 초과 시 복잡
- 화살표 교차 최소화
- 레인 높이 균등하게

## 11. 매트릭스 그리드 (Strategic Map) [신규]

**적합한 개념**: 2x2/3x3 전략 맵, 포지셔닝, 우선순위 결정, 리스크 매트릭스

### 구조
- X축: 한 차원 (예: 영향도)
- Y축: 다른 차원 (예: 긴급도)
- 사분면/구분면: 각 조합 영역

### ASCII 예시 (2x2)
```
     높음 ▲
          │  ┌─────────────┬─────────────┐
          │  │             │             │
     영   │  │   우선순위  │   즉시     │
     향   │  │     낮음    │   실행     │
     도   │  │             │             │
          │  ├─────────────┼─────────────┤
          │  │             │             │
          │  │    보류     │   계획     │
          │  │   (제외)    │   수립     │
          │  │             │             │
     낮음 │  └─────────────┴─────────────┘
          └────────────────────────────────►
                낮음              높음
                      긴급도
```

### ASCII 예시 (3x3)
```
          │    낮음    │    중간    │    높음    │
──────────┼────────────┼────────────┼────────────┤
   높음   │     C      │     B      │     A      │
──────────┼────────────┼────────────┼────────────┤
   중간   │     D      │     C      │     B      │
──────────┼────────────┼────────────┼────────────┤
   낮음   │     E      │     D      │     C      │
──────────┴────────────┴────────────┴────────────┘
```

### 시각화 원칙
- 축 라벨 명확하게 표시
- 사분면별 색상 구분
- 핵심 영역 강조
- 항목 배치로 포지셔닝 표현

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 그리드 크기 | 2x2 또는 3x3 |
| 축 라벨 글자수 | 8자 이내 |
| 영역 제목 글자수 | 10자 이내 |
| 배치 항목 수 | 4-12개 |
| 텍스트 밀도 | 중간 |
| 최소 폰트 크기 | 14pt |

### 색상 배치 전략
- **메인**: 핵심 사분면 (예: 즉시 실행)
- **서브**: 중요 사분면
- **강조**: 특정 포지션 항목
- **배경**: 낮은 우선순위 영역

### 적합 케이스
- 우선순위 매트릭스
- 리스크 평가
- 기술 포트폴리오 분석

### 부적합 케이스
- 프로세스 흐름
- 시간 기반 정보

### 주의사항
- 4x4 이상은 복잡해짐
- 축 정의 명확하게
- 항목 겹침 방지

## 12. Funnel/깔때기 [신규]

**적합한 개념**: 단계별 축소, 선별 과정, 전환율, 파이프라인

### 구조
- 상단: 가장 넓은 입력 (모집단)
- 중간: 점차 좁아지는 단계들
- 하단: 최종 결과 (가장 좁음)

### ASCII 예시
```
        ┌───────────────────────────────────┐
        │         1,000건 접수              │
        │              (100%)               │
        └─────────────────┬─────────────────┘
              ┌───────────┴───────────┐
              │     500건 1차 심사     │
              │         (50%)         │
              └───────────┬───────────┘
                    ┌─────┴─────┐
                    │ 100건 선정 │
                    │   (10%)   │
                    └─────┬─────┘
                       ┌──┴──┐
                       │ 50건 │
                       │최종  │
                       └─────┘
                        (5%)
```

### 시각화 원칙
- 위에서 아래로 자연스럽게 좁아짐
- 각 단계 수치/비율 표시
- 단계별 색상 변화 (연함→진함)
- 화살표로 흐름 표현

### 권장 사양

| 항목 | 권장값 |
|------|--------|
| 단계 수 | 3-5개 |
| 단계 라벨 글자수 | 12자 이내 |
| 수치 표시 | 필수 |
| 텍스트 밀도 | 낮음 |
| 최소 폰트 크기 | 16pt |

### 색상 배치 전략
- **메인**: 최종 결과 단계
- **서브**: 중간 단계들 (그라데이션)
- **강조**: 핵심 전환 포인트
- **배경**: 초기 단계 (가장 연한 색)

### 적합 케이스
- 사업화 파이프라인
- 선정 프로세스
- 고객 전환 퍼널

### 부적합 케이스
- 병렬 프로세스
- 순환 구조

### 주의사항
- 비율 표시로 축소 정도 명확히
- 단계 간 높이 균등하게
- 최하단 너무 좁으면 텍스트 안 보임
