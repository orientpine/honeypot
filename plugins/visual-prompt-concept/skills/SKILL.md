# 개념 프롬프트 생성기 (Visual Prompt Concept)

핵심 개념을 직관적으로 시각화하는 TED 스타일 이미지 생성 프롬프트를 작성합니다.
**한 이미지, 한 메시지** 원칙으로 독자의 즉각적 이해를 돕는 시각 자료를 만듭니다.

---

## visual-prompt-gov와의 차이

| 항목 | visual-prompt-gov | visual-prompt-concept |
|------|-------------------|----------------------|
| 스타일 | 정부/공공기관 보고서 | TED/교육 프레젠테이션 |
| 정보량 | 다량 (25개 요소) | 최소 (10개 이하) |
| 시각화 | 복잡한 인포그래픽 | 직관적 시각 메타포 |
| 3D 스타일 | 산업용 렌더링 | 개념적/추상적 렌더링 |
| 텍스트 | 상세 라벨링 | 핵심 키워드만 |
| 목적 | 정보 전달 | 개념 각인 |

---

## 사용자 입력 스키마

### 필수 입력

| 항목 | 설명 | 예시 |
|------|------|------|
| 핵심 개념 | 시각화할 개념/주제 | "머신러닝의 학습 과정", "블록체인 합의 메커니즘" |

### 선택 입력

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 시각화 유형 | 선호하는 시각화 방식 | 자동 선택 |
| 무드 테마 | 5가지 무드 중 선택 | `clarity` (명료) |
| 출력 폴더 | 프롬프트 저장 위치 | `concept_image_gen/prompts/` |
| 슬라이드 수 | 생성할 이미지 개수 | 1 |

## 워크플로우

### Phase 1: 개념 분석

```
+-- Step 1-1. 핵심 개념 파악
|   +-- 사용자 입력에서 핵심 개념 추출
|   +-- 개념의 본질적 특성 식별
|   +-- 시각화 가능한 요소 도출
|
+-- Step 1-2. 시각적 메타포 탐색
|   +-- 개념을 대변할 수 있는 시각적 비유 탐색
|   +-- 일상적 경험과 연결되는 이미지 도출
|   +-- 직관적 이해를 돕는 구조 선택
|
+-- Step 1-3. 핵심 메시지 정의
    +-- 한 문장으로 전달할 메시지 정의
    +-- 이미지를 본 후 기억할 한 가지
    +-- 불필요한 정보 제거
```

### Phase 2: 무드 테마 선택

```
+-- Step 2-1. 개념 성격 분석
|   +-- 기술적/과학적 → tech-focus
|   +-- 성장/발전 → growth
|   +-- 연결/관계 → connection
|   +-- 명확/단순 → clarity
|   +-- 혁신/변화 → innovation
|
+-- Step 2-2. 테마 색상 로드
    +-- references/prompt_style_guide.md에서 테마 색상 로드
    +-- 주조색, 보조색, 강조색 확정
```

**무드 테마 옵션:**

| 테마 ID | 테마명 | 적합 개념 | 주조색 |
|:---|:---|:---|:---|
| `clarity` | 명료 | 설명, 정의, 구조 | #2D3436 |
| `tech-focus` | 테크 | 기술, AI, 데이터 | #0984E3 |
| `growth` | 성장 | 발전, 학습, 진화 | #00B894 |
| `connection` | 연결 | 네트워크, 관계, 협력 | #6C5CE7 |
| `innovation` | 혁신 | 변화, 창조, 돌파 | #E17055 |

### Phase 3: 시각화 유형 선택

[references/layout_types.md](references/layout_types.md) 참조하여 개념에 맞는 시각화 선택:

| 개념 유형 | 시각화 유형 |
|:---|:---|
| 과정, 변환 | 플로우 메타포 |
| 구조, 계층 | 구조 메타포 |
| 관계, 연결 | 네트워크 메타포 |
| 대비, 차이 | 대비 메타포 |
| 성장, 발전 | 진화 메타포 |
| 핵심, 중심 | 중심 메타포 |
| 순환, 반복 | 순환 메타포 |
| 분류, 종류 | 그룹 메타포 |

### Phase 4: 프롬프트 작성

[references/prompt_style_guide.md](references/prompt_style_guide.md)의 스타일 가이드 적용.
[assets/output_template/prompt_template.md](assets/output_template/prompt_template.md) 템플릿 기반 작성.

```
+-- Step 4-1. 프롬프트 초안 작성
|   +-- 메타 섹션 (1~6): 영어로 작성
|   +-- 콘텐츠 섹션 (7~9): 한글로 작성
|   +-- 선택된 테마 색상 적용
|
+-- Step 4-2. 미니멀 검증
|   +-- 텍스트 요소 10개 이하
|   +-- 핵심 메시지 1개만 전달
|   +-- 불필요한 장식 요소 제거
|
+-- Step 4-3. 임팩트 검증
    +-- 시각적 메타포 명확성
    +-- 즉각적 이해 가능 여부
    +-- 기억에 남는 구성인지
```

**프롬프트 구조:**

```
[INSTRUCTION - DO NOT RENDER IN IMAGE]

1. Purpose (영어)
2. Visual Concept (영어 - 시각적 메타포)
3. Style (영어)
4. Colors (영어 - 선택된 테마 색상)
5. Lighting & Mood (영어)
6. Resolution (영어)

[콘텐츠 - 한글]

7. 핵심 메시지
8. 시각 구성
9. 텍스트 요소 (최소)
10. Forbidden Elements (영어)
11. 최종 결과물 목표
```

### Phase 5: 출력

```
+-- Step 5-1. 출력 폴더 생성
|   +-- concept_image_gen/prompts/ 폴더 생성 (없으면)
|
+-- Step 5-2. 프롬프트 파일 저장
|   +-- 파일명: [번호]_[개념명].md
|   +-- 인코딩: UTF-8
|
+-- Step 5-3. 인덱스 파일 생성
    +-- prompt_index.md 생성
    +-- 생성된 프롬프트 목록 기록
```

## 핵심 원칙 (TED 스타일)

### 1. One Slide, One Message

```
- 한 이미지에 하나의 핵심 메시지만
- 모든 요소가 그 메시지를 강화
- 불필요한 정보는 과감히 제거
```

### 2. Visual Metaphor First

```
- 추상적 개념을 구체적 이미지로
- 일상에서 익숙한 비유 활용
- 텍스트 없이도 이해 가능한 구성
```

### 3. Minimal but Impactful

```
- 텍스트 요소 최대 10개
- 여백은 의도적으로 확보
- 단순함 속의 풍부함 추구
```

### 4. 3D or Icon - Context Dependent

```
- 물리적/기계적 개념 → 3D 렌더링
- 추상적/프로세스 개념 → 아이콘/도형
- 혼합 사용 시 일관성 유지
```

## 텍스트 규격 (필수)

**미니멀 원칙: 적을수록 좋다**

| 요소 | 최소 크기 | 권장 크기 | 최대 글자수 |
|:---|:---:|:---:|:---:|
| 핵심 메시지 | **36pt** | 48-72pt | 10자 |
| 보조 라벨 | **18pt** | 20-24pt | 8자 |
| 키워드 | **16pt** | 18-20pt | 6자 |
| 캡션/주석 | **사용 금지** | - | - |

**텍스트 밀도 제한:**
- 이미지 전체 텍스트 요소: **최대 10개**
- 한 영역 내 라벨: **최대 3개**
- 문장 사용 금지, 키워드만

---

## 공통 규칙

- **텍스트**: 한글 기본, 핵심 용어는 영어 병기 가능
- **스타일**: 개념적 3D 또는 플랫 아이콘 (주제에 따라)
- **시점**: 정면 또는 약한 등각(Isometric)
- **배경**: 단색 또는 미니멀 그라데이션
- **금지**: 복잡한 인포그래픽, 과도한 텍스트, 장식 요소
- **해상도**: 8K Quality
